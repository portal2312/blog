이 섹션에는 Docker와 더 잘 작동하도록 Linux 호스트를 구성하기위한 선택적인 절차가 포함되어 있습니다.

Manage Docker as a non-root user

Docker를 비 루트 사용자로 관리

docker 데몬은 TCP 포트 대신 Unix 소켓에 바인드합니다. 기본적으로 Unix 소켓은 root 사용자가 소유하고 다른 사용자는 sudo를 사용해서만 액세스 할 수 있습니다. docker daemon 은 항상 root 사용자로 실행됩니다.

docker 명령을 사용할 때 sudo를 사용하고 싶지 않으면 docker라는 유닉스 그룹을 만들어 사용자를 추가하십시오. docker 데몬이 시작되면 docker 그룹이 UNIX 소켓의 소유권을 읽고 쓸 수 있게 만듭니다.

Warning: docker 그룹은 루트 사용자와 동일한 권한을 부여합니다. 이것이 시스템의 보안에 미치는 영향에 대한 자세한 내용은 Docker Daemon Attack Surface를 참조하십시오.

docker 그룹 생성하고 사용자 추가하기:

  1. docker group 생성하기.

    $ sudo groupadd docker

  2. docker 그룹에 사용자 추가하기.

    $ sudo usermod -aG docker $USER

  3. 로그아웃하고 다시 로그인하여 그룹 구성원을 다시 갱신하십시오.

가상 컴퓨터에서 테스트를 수행하는 경우 변경 내용을 적용하려면 가상 컴퓨터를 다시 시작해야 할 수 있습니다. X Windows와 같은 데스크탑 Linux 환경에서 세션을 완전히 로그아웃 한 다음 다시 로그인하기.

  1. sudo 없이 docker 명령어를 실행할 수 있는지 검증하기.

    $ docker run hello-world

    이 명령은 테스트 이미지를 다운로드하여 컨테이너에서 실행합니다. 컨테이너가 실행되면 정보 메시지를 인쇄하고 종료합니다.

    처음에 docker 그룹에 사용자를 추가하기 전에 sudo를 사용하여 Docker CLI 명령을 실행 한 경우 ~/.docker/ 디렉토리가 sudo 명령으로 인해 잘못된 권한으로 생성되었음을 나타내는 다음 오류가 표시될 수 있습니다.

    WARNING: Error loading config file: /home/user/.docker/config.json -
stat /home/user/.docker/config.json: permission denied

    이 문제를 수정하려면, ~/.docker/ 디렉터리(자동으로 다시 만들어 지지만 모든 사용자 지정 설정이 손실됩니다)를 삭제한다, 또는 이 소유권한 과 승인권한을 변경하는데 다음과 같은 명령어를 사용하기:

    $ sudo chown "$USER":"$USER" /home/"$USER"/.docker -R
$ sudo chmod g+rwx "/home/$USER/.docker" -R

Configure Docker to start on boot

Docker 시동시 시작 설정하기

현재의 대부분의 Linux 배포판 (RHEL, CentOS, Fedora, Ubuntu 16.04 이상)은 systemd 를 사용하여 시스템 부팅시 시작될 서비스를 관리합니다. Ubuntu 14.10 이하는 upstart 사용합니다.

systemd 

$ sudo systemctl enable docker

사용하지 않을 경우:

$ sudo systemctl disable docker

HTTP 프록시를 추가해야하는 경우 Docker 런타임 파일에 대해 다른 디렉토리 나 파티션을 설정하거나 다른 사용자 정의를 수행하려면 systemd Docker 데몬 옵션 사용자 정의를 참조하십시오.

upstart

Docker는 upstart를 사용하여 부팅 할 때 자동으로 시작되도록 구성됩니다. 이 동작을 사용하지 않으려면 다음 명령을 사용하십시오.

$ echo manual | sudo tee /etc/init/docker.override

chkconfig 

$ sudo chkconfig docker on

Use a different storage engine

다은 저장 엔진 사용하기

다른 저장소 엔진에 대한 자세한 내용은 저장소 드라이버를 참조하십시오. 기본 스토리지 엔진과 지원되는 스토리지 엔진 목록은 호스트의 Linux 배포 및 사용 가능한 커널 드라이버에 따라 다릅니다.

Reference

https://docs.docker.com/install/linux/linux-postinstall/
