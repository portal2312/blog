Prerequisites

Introduction

이 전체 자습서에 대해 동일한 작성 파일을 편집하고 있습니다. 음, 좋은 소식이 있습니다. 그 작곡 파일은 당신의 컴퓨터에서와 마찬가지로 프로덕션에서 잘 작동합니다. 여기서는 Dockerized 응용 프로그램을 실행하기 위한 몇 가지 옵션을 살펴 보겠습니다.

Choose an option

Docker CE (Cloud provider)

프로덕션 환경에서 Docker Community Edition을 사용해도 괜찮 으면 Docker Cloud를 사용하여 Amazon Web Services, DigitalOcean 및 Microsoft Azure와 같은 유명한 서비스 공급자에서 응용 프로그램을 관리 할 수 ​​있습니다.

설정 및 배포:

  • Docker Cloud에 원하는 공급자를 연결하여 Docker Cloud에 자동으로 VM을 프로비저닝하고 "Dockerize"권한을 부여합니다.
  • Docker Cloud를 사용하여 컴퓨팅 리소스를 만들고 swarm 을 만들 수 있습니다.
  • app 을 배포하십시오.

Note: 여기서 Docker Cloud 문서에 링크 될 것입니다. 각 단계를 완료 한 후에 이 페이지로 다시 돌아오십시오.

Connect Docker Cloud

Docker Cloud는 standard mode 나 Swarm mode 에서 실행할 수 있습니다.

Docker Cloud를 standard mode 로 실행중인 경우 아래 지침에 따라 서비스 제공 업체를 Docker Cloud 에 연결하십시오.

  • Amazon Web Services 설치 안내서
  • DigitalOcean 설치 안내서
  • Microsoft Azure 설치 가이드
  • 패킷 설치 안내서
  • SoftLayer 설정 가이드
  • Docker Cloud Agent를 사용하여 자신의 host 가져오기

Swarm mode (Amazon Web Services 또는 Microsoft Azure에 권장)로 실행중인 경우 다음 섹션으로 넘어가서 create your swarm 을 참조하십시오.

Create your swarm

swarm 생성 준비:

  • Amazon Web Services (AWS)를 사용 중이라면 자동으로 AWS에 swarm을 생성 할 수 있습니다.
  • Microsoft Azure를 사용하는 경우 Azure에 자동으로 swarm을 만들 수 있습니다.
  • 그렇지 않으면 Docker Cloud UI에서 노드를 만들고 Docker Cloud를 통해 SSH를 통해 4 부에서 배운 docker swarm init 및 docker swarm join 명령을 실행하십시오. 마지막으로 화면 맨 위에서 토글을 클릭하여 swarm mode 를 활성화하고 방금 생성 한 swarm 을 등록하십시오.

Note: Docker Cloud Agent를 사용하여 자체 호스트를 가져 오는 경우이 공급자는 스웜 모드를 지원하지 않습니다. Docker Cloud로 기존 스웜을 등록 할 수 있습니다.

Deploy your on a cloud provider

  1. Docker Cloud를 통해 swarm와 연결하십시오. 연결할 수있는 몇 가지 방법이 있습니다.
- Swarm 모드의 Docker Cloud 웹 인터페이스에서 페이지 상단의 Swarms를 선택하고 연결할 Swarm을 클릭 한 다음 주어진 명령을 명령 행 터미널에 복사하여 붙여 넣으십시오.

`그림`

또는

- Docker for Mac 또는 Docker for Windows에서는 데스크톱 응용 프로그램 메뉴를 통해 직접 Swarm에 연결할 수 있습니다.

`그림`

어느 쪽이든, 이것은 context 가 로컬 컴퓨터이지만 Docker 명령이 클라우드 서비스 공급자에서 실행중인 swarm 에 라우팅되는 터미널을 엽니다.
로컬 파일 시스템과 원격 swarm에 직접 액세스하여 순수 docker 명령을 사용할 수 있습니다.

  1. 실행 docker stack deploy -c docker-compose.yml getstartedlab 클라우드 host swarm 에 배포합니다.

    docker stack deploy -c docker-compose.yml getstartedlab

Creating network getstartedlab_webnet
Creating service getstartedlab_web
Creating service getstartedlab_visualizer
Creating service getstartedlab_redis

    app 은 이제 클라우드 재공자에서 지금 동작하고 있는 중이다.

RUN SOME SWARM COMMANDS TO VERIFY THE DEPLOYMENT

이미 완료된 것처럼 swarm 명령 줄을 사용하여 swarm을 찾아보고 관리 할 수 ​​있습니다. 지금까지 익숙해 져야 할 몇 가지 예가 있습니다.

  • docker node ls 사용하여 node 목록을 본다.

    $ docker node ls

  • docker service ls 사용하여 service 목록을 본다.

    $ docker service ls

  • docker service ps <service> 사용하여 서비스에 대한 작업을 봅니다.

    $ docker service ps vy7n2piyqrtr

OPEN PORTS TO SERVICES ONB CLOUD PROVIDER MACHINES

이 시점에서 app 은 방금 실행 한 docker 명령에 의해 입증된 것처럼 클라우드 제공자 서버에 swarm 으로 배포됩니다. 그러나 다음과 같이 클라우드 서버의 port 를 열어야합니다:

  • worker nodes 에서 redis 서비스와 web 서비스 간의 통신 허용
  • worker nodes 에서 web 서비스로의 인바운드 트래픽을 허용하여 Hello World 및 Visualizer가 웹 브라우저에서 액세스 할 수 있도록합니다.
  • manager 를 실행중인 서버에서 인바운드 SSH 트래픽을 허용합니다 (클라우드 공급자에 이미 설정되었을 수 있음)

다음은 각 서비스에 대해 노출해야하는 포트입니다:

Service Type Protocol Port
web HTTP TCP 80
visualizer HTTP TCP 8080
redis TCP TCP 6379

이 작업을 수행하는 방법은 클라우드 제공 업체에 따라 다릅니다.

예를 들어 Amazon Web Services (AWS)를 사용합니다.

What about the redis service to persist data?

redis 서비스가 작동하려면 관리자가 실행중인 클라우드 서버로 ssh하고 docker 스택을 배포하기 전에 /home/docker/ 에 data/ 디렉토리를 만들어야합니다. 또 다른 옵션은 docker-stack.yml 의 데이터 경로를 manager 서버의 기존 경로로 변경하는 것입니다. 이 예제는이 단계를 포함하지 않으므로 예제 출력에서 ​​redis 서비스가 작동하지 않습니다.

Iteration and cleanup

여기에서 이전 튜토리얼에서 배웠던 모든 것을 할 수 있습니다.

  • docker-compose.yml 파일을 변경하고 docker stack deploy 명령으로 on-the-fly를 재배포하여 응용 프로그램을 캐일합니다.

  • 코드를 편집하여 응용 프로그램 동작을 변경한 다음 새 이미지를 다시 작성하고 푸시합니다. (이렇게 하려면 이전에 앱을 제작하고 이미지를 게시하는 데 사용한 것과 동일한 단계를 따르십시오).

  • docker stack rm 을 사용하여 stack 을 삭제할 수 있습니다. 예: bash $ docker stack rm getstartedlab

local Docker 컴퓨터 VM에서 swarm 을 실행하는 시나리오와 달리, swarm 및 여기에 배포된 모든 응용 프로그램은 로컬 호스트를 종료했는지 여부에 관계없이 클라우드 서버에서 계속 실행됩니다.

Congratulations

전체 Docker 플랫폼에 대해 완전한 stack 으로 개발 완료된 둘러보기를 수행했습니다.

Docker 플랫폼에는 여기에 설명된 것보다 훨씬 많은 것이 있지만 컨테이너, 이미지, 서비스, 스웜, 스택, 확장,로드 균형 조정, 볼륨 및 배치 제약에 대한 기본 개념을 잘 알고 있습니다.

더 깊게 가고 싶습니까? 다음은 권장 할만한 자료입니다.

  • User Guide : 이 사용자 가이드에는 네트워킹 및 저장 장치에 대해 설명하는 몇 가지 예가 여기에서 다뤘습니다.
  • Admin Guide : Dockerized 프로덕션 환경을 관리하는 방법을 다룹니다.
  • Training : 실습과 가상 교실 환경을 제공하는 공식 Docker 코스입니다.
  • Blog : 최근에 Docker와 관련된 일들을 다루고 있습니다.
