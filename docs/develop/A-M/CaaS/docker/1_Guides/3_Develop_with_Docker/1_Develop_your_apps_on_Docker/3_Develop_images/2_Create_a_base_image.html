기본 이미지 만들기

대부분의 Dockerfiles 는 부모 이미지에서 시작합니다. 이미지의 내용을 완전히 제어해야 하는 경우 대신 기본 이미지를 만들어야 할 수도 있습니다. 차이점은 다음과 같습니다.

  • 상위 이미지는 이미지의 기반이되는 이미지입니다. Dockerfile 에서 FROM 지시문의 내용을 참조합니다. Dockerfile의 각 후속 선언은 이 상위 이미지를 수정합니다. 대부분의 Dockerfiles는 기본 이미지가 아닌 부모 이미지에서 시작합니다. 그러나 이 용어는 때로는 서로 바꿔 사용할 수 있습니다.

  • 기본 이미지는 Dockerfile에 FROM 행이 없거나 FROM scratch 가 있습니다.

이 주제에서는 기본이 이미지를 만드는 몇 가지 방법을 보여줍니다. 특정 프로세스는 패키지하려는 Linux 배포판에 크게 의존합니다. 아래에 몇 가지 예가 나와 있으며, 새로운 요청을 제출하도록 요청을 제출하는 것이 좋습니다.

Create a full image using tar

tar를 사용하여 전체 이미지 만들기

일반적으로 부모 이미지로 패키징하고자 하는 배포판을 실행중인 작업 머신으로 시작하는 것이 좋지만 Debian의 Debootstrap 과 같은 일부 도구에는 필요하지 않지만 Ubuntu 이미지를 빌드시 사용할 수 있습니다.

Ubuntu 부모 이미지를 만드는 것이 이렇게 간단 할 수 있습니다:

$ sudo debootstrap xenial xenial > /dev/null
$ sudo tar -C xenial -c . | docker import - xenial

a29c15f1bf7a

$ docker run xenial cat /etc/lsb-release

DISTRIB_ID=Ubuntu
DISTRIB_RELEASE=16.04
DISTRIB_CODENAME=xenial
DISTRIB_DESCRIPTION="Ubuntu 16.04 LTS"

Docker GitHub Repo에서 부모 이미지를 만들기위한 예제 스크립트가 더 있습니다:

  • [BusyBox]
  • CentOS/Scientific Linux CERN (SLC) on [Debian/Ubuntu] or on [CentOS/RHEL/SLC/etc].
  • [Debian/Ubuntu]

Create a simple parent image using scratch

scratch를 사용하여 간단한 부모 이미지 만들기

Docker의 reserved, minimal image, scratch 를 컨테이너 작성의 시작점으로 사용할 수 있습니다. scratch "image" 신호를 빌드 프로세스에 사용하면 Dockerfile의 다음 명령이 이미지의 첫 번째 파일 시스템 레이어가 되도록합니다.

scratch 가 허브의 Docker 저장소에 나타나지만 끌어서 놓거나 실행하거나 scratch라는 이름의 이미지에 태그를 추가 할 수 없습니다. 대신 Dockerfile 에서 참조할 수 있습니다. 예를 들어, scratch를 사용하여 최소 컨테이너를 생성하려면 다음을 수행하십시오:

FROM scratch
ADD hello /
CMD ["/hello"]

Docker GitHub 예제 C 소스 코드에서 "hello"실행 가능 예제를 빌드하고 -static flag 로 컴파일 한 경우 다음을 사용하여 이 Docker 이미지를 빌드 할 수 있습니다. docker build --tag hello .

Note: Mac 용 Docker 및 Windows 용 Docker는 Linux VM을 사용하므로 Linux 툴 체인을 사용하여이 코드를 컴파일하여 Linux 바이너리로 끝내야합니다. 걱정하지 마라. 당신은 신속하게 리눅스 이미지와 빌드 환경을 풀어 그 안에 빌드 할 수있다.

$ docker run --rm -it -v $PWD:/build ubuntu:16.04
container# apt-get update && apt-get install build-essential
container# cd /build
container# gcc -o hello -static -nostartfiles hello.c

다음을 사용하여 (Linux, Mac 또는 Windows에서) 실행할 수 있습니다. docker run --rm hello

이 예제는 자습서에서 사용되는 hello-world 이미지를 만듭니다. 테스트 해보고 싶다면 저장소를 복제 할 수 있습니다.

추가 리소스

Dockerfile 을 작성하는 데 도움이되는 더 많은 리소스가 있습니다.

  • 참조 섹션의 Dockerfile 에서 사용할 수있는 [모든 지침에 대한 완전한 가이드]가 있습니다.
  • 명확하고 읽기 쉽고 유지 보수가 쉬운 Dockerfile 을 작성할 수 있도록 Dockerfile [모범 사례 가이드]를 작성했습니다.
  • 새로운 공식 저장소를 만드는 것이 목표라면 Docker의 [공식 저장소]를 반드시 읽으십시오.

Reference

https://docs.docker.com/engine/userguide/eng-image/baseimages/
