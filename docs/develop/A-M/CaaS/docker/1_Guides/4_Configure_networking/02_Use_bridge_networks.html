네트워킹 측면에서 브리지 네트워크는 네트워크 세그먼트간에 트래픽을 전달하는 링크 계층 장치입니다. 브리지는 호스트 시스템의 커널에서 실행되는 하드웨어 장치 또는 소프트웨어 장치 일 수 있습니다.

Docker와 관련하여 브리지 네트워크는 소프트웨어 브리지를 사용하여 같은 브리지 네트워크에 연결된 컨테이너가 해당 브리지 네트워크에 연결되지 않은 컨테이너로부터 격리하면서 통신할 수 있도록 합니다.

Docker 브리지 드라이버는 서로 다른 브리지 네트워크의 컨테이너가 서로 직접 통신 할 수 없도록 호스트 시스템에 규칙을 자동으로 설치합니다.

브리지 네트워크는 동일한 Docker 데몬 호스트에서 실행되는 컨테이너에 적용됩니다.

다른 Docker 데몬 호스트에서 실행되는 컨테이너 간의 통신을 위해 OS 수준에서 라우팅을 관리하거나 를 사용할 수 [overlay network]있습니다.

Differences between user-defined bridges and the default bridge

사용자 정의 브릿지와 기본 브릿지 사이의 다른점

  • user-defined bridge는 컨테이너화 된 응용 프로그램 간의보다 나은 격리 및 상호 운용성을 제공합니다.

  • 사용자 정의 브리지는 컨테이너간에 자동 DNS 분석을 제공합니다.

  • 컨테이너는 사용자 정의 네트워크에서 즉시 부착 및 분리 할 수 ​​있습니다.

  • 각 사용자 정의 네트워크는 구성 가능한 브리지를 만듭니다.

  • 기본 브리지 네트워크의 연결된 컨테이너는 환경 변수를 공유합니다.

Manage a user-defined bridge

docker network create 명령어를 이용하여 user-defined bridge 만들기

$ docker network create my-net

서브넷, IP 주소 범위, 게이트웨이 및 기타 옵션을 지정할 수 있습니다.

자세한 내용은 docker network create 참조 또는 docker network create --help의 출력을 참조하십시오.

docker network rm 명령을 사용하여 사용자 정의 브리지 네트워크를 제거합니다.

container 가 현재 네트워크에 연결되어 있으면 먼저 연결을 끊습니다.

$ docker network rm my-net

Note: What's really happening?

user-defined bridge 를 생성 또는 제거하거나 사용자 정의 브리지에서 컨테이너를 연결 또는 연결 해제 할 때 Docker는 운영 체제 관련 도구를 사용하여 기본 네트워크 인프라를 관리합니다 (예: 브리지 장치 추가 또는 제거 또는 iptables 규칙 구성 리눅스). 이러한 세부 사항은 구현 세부 사항으로 간주되어야합니다. Docker가 user-defined networks 를 관리하도록하십시오.

Connect a container to a user-defined bridge

새 container 를 작성할 때 하나 이상의 --network flag를 지정할 수 있습니다.

이 예제는 Nginx 컨테이너를 my-net 네트워크에 연결합니다.

또한 컨테이너의 포트 80을 Docker 호스트의 포트 8080에 게시하므로 외부 클라이언트가 해당 포트에 액세스 할 수 있습니다.

my-net 네트워크에 연결된 다른 컨테이너는 my-nginx 컨테이너의 모든 포트에 액세스 할 수 있으며 반대의 경우도 마찬가지입니다.

$ docker create --name my-nginx \
  --network my-net \
  --publish 8080:80 \
  nginx:latest

실행중인 컨테이너를 기존 사용자 정의 브리지에 연결하려면 docker network connect 명령을 사용하십시오. 다음 명령은 이미 실행중인 my-nginx 컨테이너를 이미 존재하는 my-net 네트워크에 연결합니다.

$ docker network connect my-net my-nginx

Disconnect a container from a user-defined bridge

실행중인 컨테이너를 사용자 정의 브리지에서 연결 해제하려면 docker network disconnect 명령을 사용하십시오. 다음 명령은 my-nginx 컨테이너를 my-net 네트워크에서 연결 해제합니다.

$ docker network disconnect my-net my-nginx

Use IPv6

Docker 컨테이너에 대해 IPv6 지원이 필요한 경우 IPv6 네트워크를 만들거나 컨테이너 IPv6 주소를 할당하기 전에 Docker 데몬에서 [옵션을 활성화]하고 해당 구성을 다시로드해야합니다.

네트워크를 만들 때 --ipv6 flag를 지정하여 IPv6을 활성화 할 수 있습니다.

기본 브리지 네트워크에서 IPv6 지원을 선택적으로 비활성화 할 수는 없습니다.

Enable forwarding from Docker containers to the outside world

Docker 컨테이너에서 외부로 전달 가능

기본적으로 기본 브리지 네트워크에 연결된 컨테이너의 트래픽은 외부로 전달되지 않습니다.

전달을 사용하려면 두 가지 설정을 변경해야합니다.

이것들은 Docker 명령이 아니며 Docker 호스트의 커널에 영향을 미칩니다.

  1. IP 전달을 허용하도록 Linux 커널을 구성하십시오.
$ sysctl net.ipv4.conf.all.forwarding=1
  1. iptables FORWARD 정책에 대한 정책을 DROP에서 ACCEPT로 변경하십시오.
$ sudo iptables -P FORWARD ACCEPT

이러한 설정은 다시 부팅 할 때까지 유지되지 않으므로 시작 스크립트에 추가해야 할 수 있습니다.

Use the default bridge network

기본 bridge 네트워크는 Docker의 기존 세부 정보로 간주되며 프로덕션 용도로 권장되지 않습니다.

구성은 수동 조작이며 technical shortcomings이 있습니다.

Connect a container to the default bridge network

--network flag를 사용하여 네트워크를 지정하지 않고 네트워크 드라이버를 지정하면 기본적으로 컨테이너가 기본 bridge 네트워크에 연결됩니다.

기본 bridge 네트워크에 연결된 컨테이너는 legacy --link flag를 사용하여 연결되어 있지 않으면 IP 주소로만 통신 할 수 있습니다.

Configure the default bridge network

기본 브리지 네트워크를 구성하려면 daemon.json에서 옵션을 지정합니다.

다음은 몇 가지 옵션이 지정된 예제 daemon.json입니다.

사용자 정의해야하는 설정만 지정하십시오.

{
  "bip": "192.168.1.5/24",
  "fixed-cidr": "192.168.1.5/25",
  "fixed-cidr-v6": "2001:db8::/64",
  "mtu": 1500,
  "default-gateway": "10.20.1.1",
  "default-gateway-v6": "2001:db8:abcd::89",
  "dns": ["10.20.1.2","10.20.1.3"]
}

Docker를 다시 시작하여 변경 사항을 적용하십시오.

Use IPv6 with the default bridge network

IPv6 용 Docker 지원 (IPv6 사용 참조)을 구성하면 기본 브리지 네트워크도 IPv6 용으로 자동 구성됩니다. 사용자 정의 브리지와는 달리 기본 브리지에서는 IPv6을 선택적으로 비활성화 할 수 없습니다.
