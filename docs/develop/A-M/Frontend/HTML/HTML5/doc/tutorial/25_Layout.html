HTML Layout Elements

웹 사이트는 종종 잡지 나 신문과 같이 여러 열에 내용을 표시합니다.

HTML5 새로운 의미론적인 웹 페이지의 다른 부분을 정의하는 element 들을 제공한다.

  • <header> - 문서 또는 섹션의 머리글을 정의합니다.
  • <nav> - 탐색 링크용 컨테이너를 정의합니다.
  • <section> - 문서안의 섹션을 정의합니다.
  • <article> - 독립적 인 자체 포함 된 article을 정의합니다.
  • <aside> - 내용과는 별도의 콘텐츠를 정의합니다. (sidebar 처럼)
  • <footer> - 문서 또는 섹션의 바닥글을 정의합니다.
  • <details> - 추가적인 세부정보를 정의합니다.
  • <summary> - <details> 요소의 제목을 정의합니다.

HTML Layout Techniques

여러 열 레이아웃을 만드는 방법에는 네 가지가 있습니다. 각 방법마다 장단점이 있습니다.

  • HTML tables
  • CSS float property
  • CSS framework
  • CSS flexbox

Which One th

HTML Tables

<table> 요소는 레이아웃 도구로 설계되지 않았습니다! <table> 요소의 목적은 표 형식의 데이터를 표시하는 것입니다. 따라서 페이지 레이아웃에 테이블을 사용하지 마십시오! 그들은 당신의 코드에 혼란을 가져올 것입니다. 몇 달 후에 사이트를 재 설계하는 것이 얼마나 힘든지 상상해보십시오.

CSS Frameworks

빠른 layout 생성을 원한다면 W3.CSS or Bootstrap 같은 framework 를 사용할 수 있다.

CSS Floats

CSS float 속성을 사용하여 전체 웹 레이아웃을 수행하는 것이 일반적입니다. float 는 배우기 쉽습니다. float 및 투명한 속성이 작동하는 방식을 기억하면됩니다.

단점: 플로팅 요소는 문서 흐름에 묶여있어 유연성에 해를 미칠 수 있습니다.

CSS Float and Clear 장에서 float에 대해 자세히 알아보십시오.

CSS Flexbox

Flexbox는 CSS3의 새로운 레이아웃 모드입니다.

flexbox를 사용하면 페이지 레이아웃이 다른 화면 크기와 다른 디스플레이 장치를 수용해야하는 경우 요소가 예상대로 작동합니다.

단점 : IE10 이하에서는 작동하지 않습니다.

Flexbox에 대한 자세한 내용은 CSS Flexbox 장을 참조하십시오.

Reference

https://www.w3schools.com/html/html_layout.asp
