### `<article>` vs `<section>` vs `<div>`

- `<article>` 독립적 컨텐츠
- `<section>` 관련 컨텐츠 구분
- `<div>` 관련없는, 내용만 묶는 역할

```html
<div>
  블로그
  <section>여행
    <article>한국</article>
    <article>미국</article>
  </section>
  <section>취미
    <article>음악</article>
    <article>영화
      <section>국내</section>
      <section>해외</section>
    </article>
  </section>
</div>

section

문서의 장(chapters), 머리글(headers), 바닥(footers) 글 또는 문서의 다른 섹션(sections)과 같은 문서의 섹션을 정의.

<section>
  <h1>WWF</h1>
  <p>The World Wide Fund for Nature (WWF) is....</p>
</section>

main

문서의 주요 내용을 지정합니다.

해당 tag의 요소의 내용은 문서에 고유해야합니다.

사이드 바, 탐색 링크, 저작권 정보, 사이트 로고 및 검색 양식과 같이 문서 전반에 걸쳐 반복되는

내용이 포함되어서는 안됩니다.

문서에 하나 이상의 해당 tag의 요소가 없어야 합니다.

해당 tag의 요소는

<article>, <aside>, <footer>, <header> 또는 <nav> 요소의 하위 항목이어서는 안됩니다.

<main>
  <h1>Web Browsers</h1>
  <p>Google Chrome, Internet Explorer are the most used browsers today.</p>
  <article>Google Chrome</article>
  <article>Internet Explorer</article>
</main>
