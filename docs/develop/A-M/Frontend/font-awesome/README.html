For Jekyll

설치는 일반적인 다운로드 방법보다 npm 이용시 해당 Font Awesome 을 관리를 할 수 있는 장점이 있다 (Start/Other Ways to Use/npm 참조).

Installation:

cd assets
npm init
npm install --save @fortawesome/fontawesome-free

Current path:

├── assets
│   ├── node_modules
│   |   └── @fortawesome
│   |       └── fontawesome-free
│   |           ├── css  # Stylesheets for Web Fonts (all.css)
│   |           ├── js  # SVG with JavaScript (all.js)
│   |           ├── less  # Less pre-processor (fontawesome.less)
│   |           ├── scss  # Sass pre-processor (fontawesome.scss)
│   |           ├── spritesa  # SVG sprites (solid.svg)
│   |           ├── svgs  # Individual SVG for each icon (individual *.svg icons)
│   |           │   ├── brands
│   |           │   ├── regular
│   |           │   └── solid
│   |           └── webfonts  # Web Font files used with CSS (See /css)
│   ├── main.scss
.   .

Using Web Fonts with CSS

<head>
  <link href="/your-path-to-fontawesome/css/all.css" rel="stylesheet"> <!--load all styles -->
</head>
<body>
  <i class="fas fa-user"></i> <!-- uses solid style -->
  <i class="far fa-user"></i> <!-- uses regular style -->
  <i class="fal fa-user"></i> <!-- uses light style -->
  <!--brand icon-->
  <i class="fab fa-github-square"></i> <!-- uses brands style -->
</body>

Using SVG with JavaScript

<head>
  <script defer src="/your-path-to-fontawesome/js/all.js"></script> <!--load all styles -->
</head>
<body>
  <i class="fas fa-user"></i> <!-- uses solid style -->
  <i class="far fa-user"></i> <!-- uses regular style -->
  <i class="fal fa-user"></i> <!-- uses light style -->
  <!--brand icon-->
  <i class="fab fa-github-square"></i> <!-- uses brands style -->
</body>

특정 Style 만 사용하기

css:

css/fontawesome.css, css/brands.css, css/solid.css

js:

js/fontawesome.js, js/brands.js, css/solid.js

Warning: 웹 글꼴의 경로를 염두에 두십시오.
/webfonts 및 /css 폴더를 동일한 디렉토리에 유지하는 것이 좋습니다.
그렇지 않은 경우 각 스타일의 CSS 파일에 언급 된 웹 글꼴의 경로를 변경해야합니다.

SASS (SCSS)

Adding Font Awesome to Your Compile

scss 폴더를 프로젝트에 복사하십시오.

그런 다음 전체 웹 글꼴 폴더를 정적 파일이 제공되는 프로젝트로 복사하십시오.

프로젝트의 scss/variables.scss를 열고 $fa-font-path 변수를 편집하여 webfonts 폴더의 위치를 ​​지정하십시오:

// $fa-font-path: "../webfonts" !default;  // old
$fa-font-path: "../assets/node_modules/@fortawesome/fontawesome-free/webfonts" !default;  // path 를 my site URL 기준으로 변경하기

Edit the main.scss file of my project:

// ...
@import "../assets/node_modules/@fortawesome/fontawesome-free/scss/fontawesome";
@import "../assets/node_modules/@fortawesome/fontawesome-free/scss/brands";
@import "../assets/node_modules/@fortawesome/fontawesome-free/scss/solid";

user.scss:

.user {
  @extend %fa-icon;
  @extend .fas;

  &:before {
    content: fa-content($fa-var-user);
  }
}

.twitter {
  @extend %fa-icon;
  @extend .fab;

  &:before {
    content: fa-content($fa-var-twitter);
  }
}

Note: font-awesome 을 복사하지 않고 사용하려면 assets/node_modules 있어야 한다.
jekyll 에서 build 시 aseets 을 기본적으로 참조한다.
이 때 node_modules/@fortawesome/fontawesome-free/scss/_variables.scss 에서 변수 $fa-font-path가 webfonts 참조한다. 이 때 외부 경로시 오류가 발생한다.

파일 구조 설명은 참조를 확인하기.
