Installation

OS: Ubuntu

sudo apt-get install nodejs

Version Check:

nodejs -v
npm -v

Packages

Search

  • Find the package in Official Web Site (recommand)

  • npm search <pkg>

Install

for use as a command line app (independent, global):

sudo npm install -g <pkg>

for programmatic use (dependent, current directory):

cd <my-project>
npm install <pkg>

Examples:

cd <my-project>
npm install underscore

# └── underscore@1.9.1  extraneous

Note: extraneous 상태
해당 package 가 프로젝트에서 관리하지 않거나 일시적으로 설치된 경우

npm init & npm install --save

for programmatic use (dependent, current directory)시 관리를 위해.

  1. 현재 프로젝트에 npm init 실행하기:

    npm init

# name: (test)  # 현재 프로젝트 명
# version: (1.0.0)
# description: desc
# entry point: (index.js) app.js
# test command: test.js
# git repository:
# keywords: application
# author: mkkim
# license: (ISC)
# About to write to ./package.json:

    실행 후 입력한 내용을 바탕으로 package.json 생성된다.
    package.json 은 현재 프로젝트 정보와 관련 npm packages 정보 관리한다.

    └── package.json

    package.json:

    {
  "name": "test",
  "version": "1.0.0",
  "description": "desc",
  "main": "app.js",
  "scripts": {
    "test": "test.js"
  },  
  "keywords": [
    "application"
  ],  
  "author": "mkkim",
  "license": "ISC"
}

    name: 현재 프로젝트 명 (간결하고 직관적인, 단 다른 모듈과 동일한 이름 피하기) 작성하기.

    version: SemVer(The semantic versioner for npm)로 분석 가능한 형태의 버전을 지정하기.

    description: 현재 프로젝트 설명 (npm search을 통해 찾을 때 도움 됨)

    main: 프로그램의 기본 진입 점(entry point)를 지정합니다. 패키지의 이름이 jquery이고, 사용자가 require('jquery')를 사용하면 진입 점의 메인 모듈에서 exports object가 반환(return)됩니다.

    script: 패키지 라이프 사이클에서 여러 번 실행되는 스크립트 명령을 포함합니다.

    keywords: 현재 프로젝트의 keywords (npm search을 통해 찾을 때 도움 됨)

    author: 제작자 이름 작성하기.

    license: 라이센스 지정하기 (Open Source Licenses)

  2. npm install --save <pkg>: 해당 package 가 현재 프로젝트에서 반드시 필요한 경우(의존성)

    cd <my-project>
npm install --save underscore

# └── underscore@1.9.1

    package.json:

    {
  "dependencies": {
    "underscore": "^1.9.1"
  }
}

    Current directory:

    ├── node_modules
│   └── underscore
│       ├── LICENSE
│       ├── README.md
│       ├── package.json
│       ├── underscore-min.js
│       ├── underscore-min.js.map
│       └── underscore.js
└── package.json

    Note: node_modules/
    현재 경로에서 NPM 으로 설치된 실제 packages.

Packages in the external package.json install

cp package.json ./my-projects
cd ./my-projects
npm install

Installed packages view

  • for use as a command line app (independent, global):

    npm list -g --depth=0

# /usr/local/lib
# ...
# ├── ws@7.1.1
# ...

    npm list -g

# /usr/local/lib
# ...
# ├─┬ ws@7.1.1
# │ └── async-limiter@1.0.1  # Part of ws@7.1.1
# ...

  • for programmatic use (dependent, current directory):

    npm list

Update

sudo npm update [-g] [<pkg>...]

Uninstall

sudo npm uninstall [-g] <pkg>

Use

for use as a command line app (independent, global)

for example app: uglify-js)

sudo npm install -g uglify-js

uglifyjs a.js -m -o a.min.js

for programmatic use (dependent, current directory)

Create the example.js file:

const _ = require('underscore');
const arr = [3,6,1,12];
console.log(_.first(arr));

Execute the example.js file on command line:

node example.js
# 3

Check the package on node mode:

> const _ = require('underscore');
undefined
> const arr = [3,6,1,12];
undefined
> console.log(_.first(arr));
3

Version Manager

NVM (Node Version Manager)

NVM 설치하기 전 충돌을 피하기 위해 기존 설치된 Node.js는 제거하기.

n

n 은 node package 이며 즉시 설치하여 빠르게 접근할 수 있다.

Exceptions

npm WARN enoent ENOENT: no such file or directory, open '*/package.json'

Reason: Does not exist package.json

Solutions:

npm init

npm ERR! extraneous: <pkg> <pkg-path>

Reason: packages.json에 의존성으로 등록되지 않고, 설치했기 때문

Solutions:

npm install --save <pkg>
