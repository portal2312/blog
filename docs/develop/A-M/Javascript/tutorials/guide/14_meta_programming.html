ECMAScript 2015에서 JavaScript에는 Proxy개체와 Reflect개체가 지원되었습니다. 이들은 기본적인 언어 조작 (예를 들어, 속성 검색, 대입 열거 함수 호출 등)에 인터럽트 동작을 사용자 정의 할 수 있습니다. 이 두 개체 덕분에 JavaScript로 메타 레벨 프로그래밍 할 수 있습니다.

Proxies(프록시)

ECMAScript 6에서 도입 된 Proxy개체는 특정 작업에 끼어 동작을 사용자 정의 할 수 있습니다. 예를 들어, 개체의 속성을 가져 봅시다:

var handler = {
  get: function(target, name){
    return name in target ? target[name] : 42;
}};
var p = new Proxy({}, handler);
p.a = 1;
console.log(p.a, p.b); // 1, 42

이 Proxy개체는 target(여기에서는 빈 객체)와 get트랩이 구현 된 handler객체를 정의하고 있습니다. 여기에서 프록시가 된 개체는 정의되지 않은 속성을 취득하려고했을 때 undefined 반환하지 않으며 대신에 숫자 42을 반환합니다.

더 많은 예제는 Proxy 참조하십시오.

용어

handler:

trap을 넣기 위한 자리 표시 용 객체.

trap:

속성에 대한 액세스를 제공하는 메소드. 운영 체제에서 트랩의 개념과 유사한 것입니다.

target:

proxy를 가상화하는 객체. 이것은 proxy 스토리지 백엔드로 자주 사용됩니다. 확장·설정 가능하지 않은 특성을 가진 개체에 대한 불변 조건(변경되지 않는 의미, 즉 개체의 의미 정보) 이 대상에 대해 확인됩니다.

invariant(불변 조건):

사용자 지정 동작을 구현할 때 변경되지 않는 의미를 불변 조건이라고 합니다. 핸들러의 불변 조건을 위반하는 경우 TypeError가 발생합니다.

Revocable Proxy(취소 가능한 프록시)

proxy.revocable() 메서드는 취소 가능한 Proxy 객체를 만드는 데 사용됩니다. 즉, proxy는 revoke 함수를 통해 취소될 수 있으며 proxy를 해제합니다. 그런 다음 프록시에 대한 조작으로 typeerror가 발생합니다.

var revocable = Proxy.revocable({}, {
  get: function(target, name) {
    return '[[' + name + ']]';
  }
});
var proxy = revocable.proxy;
console.log(proxy.foo); // "[[foo]]"

revocable.revoke();

console.log(proxy.foo);  // TypeError is thrown
proxy.foo = 1;           // TypeError again
delete proxy.foo;        // still TypeError
typeof proxy;            // "object", typeof doesn't trigger any trap

Reflection

Reflect는 차단 가능한 JavaScript 작업을 위한 메서드를 제공하는 기본 제공 개체입니다. 메소드는 proxy 핸들러의 메소드와 동일합니다. Reflect는 함수 객체가 아닙니다.

Reflect는 기본 작업을 처리기에서 대상으로 전달하는 데 도움이 됩니다.

예를 들어 Reflect.has()를 사용하면 in 연산자를 함수로 사용할 수 있습니다:

Reflect.has(Object, 'assign'); // true

A better apply function

ES5에서는 일반적으로 Function.prototype.apply() 메서드를 사용하여 지정된 값과 인수(배열 또는 유사한 객체)로 제공된 함수를 호출합니다.

Function.prototype.apply.call(Math.floor, undefined, [1.75]);

Reflect.apply를 사용하면 이해가 덜 쉽고 이해하기 쉬워집니다.

Reflect.apply(Math.floor, undefined, [1.75]);
// 1;

Reflect.apply(String.fromCharCode, undefined, [104, 101, 108, 108, 111]);
// "hello"

Reflect.apply(RegExp.prototype.exec, /ab/, ['confabulation']).index;
// 4

Reflect.apply(''.charAt, 'ponies', [3]);
// "i"

Checking if property definition has been successful(속성 정의가 성공적인지 확인하기)

성공하면 Object를 반환하는 Object.defineProperty를 사용하고 그렇지 않으면 TypeError를 throw합니다. try...catch 블록을 사용하여 속성을 정의하는 동안 발생한 오류를 catch합니다. Reflect.defineProperty는 Boolean success 상태를 반환하기 때문에 여기서 if...else 블록을 사용할 수 있습니다.

if (Reflect.defineProperty(target, property, attributes)) {
  // success
} else {
  // failure
}
