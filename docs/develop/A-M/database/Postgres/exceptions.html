Sorting 안되는 현상

Database Collate 가 en_US.UTF-8 인 경우 한글 정렬이 안되는 현상.

postgres 에서 Databse 보기:

postgres=# \l

   Name    |  Owner   | Encoding |   Collate   |    Ctype    |   Access privileges
-----------+----------+----------+-------------+-------------+-----------------------
 ktmss2    | postgres | UTF8     | ko_KR.UTF8  | ko_KR.UTF8  |
 postgres  | postgres | UTF8     | en_US.UTF-8 | en_US.UTF-8 |
 template0 | postgres | UTF8     | en_US.UTF-8 | en_US.UTF-8 | =c/postgres          +
           |          |          |             |             | postgres=CTc/postgres
 template1 | postgres | UTF8     | en_US.UTF-8 | en_US.UTF-8 | postgres=CTc/postgres+

일시적인 정렬하는 SELECT 명령어:

select name from "USER" order by name collate "C";

schema 추가하기:

ALTER TABLE "USER" ALTER COLUMN name TYPE character varying(32) COLLATE postgres;

SELECT name FROM "USER" ORDER BY name ASC;

Database 에 적용하기:

update
  pg_database set encoding = 6,
  datcollate = 'ko_KR.UTF8',
  datctype = 'ko_KR.UTF8'
where
  datname = 'ktmss2';

https://www.postgresql.org/docs/9.3/static/sql-altertable.html

https://lauris.github.io/database/alter-column-differences-between-mysql-postgresql

UPDATE excute read-only transaction

web page 에서 error msg 출력됨.

Resolve:

pgpool 사용시 /opt/hts/netcruz/webapps/nom/nom/settings.py DB_PORT: 5432 -> 9999 해결.

postgresql error msg: cannot set transaction read-write mode during recovery

현재 nom 단일망 5432 설정, pgpool 9999 또는 pgbounce 6432 설정

/home/postgres/data/recovery.conf:

standby_mode          = 'on'
primary_conninfo      = 'host=nom-master port=5432 user=postgres'
trigger_file          = '/var/log/pgpool/trigger/trigger'   <<< 이 파일을 참조 (*단일망 구성해도 반드시 있어야 함)

Resolve:

사용하지 않는 pgpool down 후 postgresql restart

alter column ERROR: column "컬럼명" contains null values

alter table "USER" alter column lock_time set default 0;
alter table "USER" alter column lock_time set not null;

select userid, lock_time from "USER";:

userid    | lock_time
-------------+------------
masteradmin |
UserBefore  |
UserAfter   |
mkkim1      |
restUser    | 1467366949
mkkim2      |
(6 rows)

Resolve:

update "USER" set lock_time = 0 where lock_time is null;
alter table "USER" alter column lock_time set not null;

userid    | lock_time
-------------+------------
mkkim2      |          0
restUser    | 1467366949
masteradmin |          0
UserBefore  |          0
UserAfter   |          0

(6 rows)
