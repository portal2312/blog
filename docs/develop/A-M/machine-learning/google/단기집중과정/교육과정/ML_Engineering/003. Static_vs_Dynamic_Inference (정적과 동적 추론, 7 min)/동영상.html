Static vs. Dynamic Inference (정적 추론과 동적 추론 비교)

Table of contents

추론 = 예측

오프라인 추론

  • MapReduce 등을 사용하여 가능한 모든 예측을 일괄적으로 생성합니다.
  • 그런 다음 예측을 SSTable 또는 Bigtable에 기록하고 캐시/조회 테이블에 입력합니다.
  • 장점
    • 추론 비용을 크게 신경 쓸 필요가 없음 (모델이 비싼 경우 계산 비용이 많이 들어가지만 머신을 몇 개 더 사용하면 됨)
    • batch quota를 사용 가능
    • 데이터 예측값을 푸시하기 전에 사후 검증이 가능 (유효성 점수 기록후 합당한지 정상성 체크 가능)
  • 단점
    • 알고 있는 항목만 예측할 수 있으며 길게 늘어지는 분포(시간에 따른 변경 영역)에 적합하지 않습니다. 사전 모든 예제 입수 가능하거나 헤드 쿼리만 처리하면 된다면 오프라인 점수 산정 참고표를 사용 가능.
    • 업데이트 지연 시간이 몇 시간 내지 며칠이 걸릴 수 있습니다.

온라인 추론

  • 서버를 사용하여 요청 시 예측합니다.
  • 장점
    • 새로 들어오는 항목을 바로 예측할 수 있으며 길게 늘어지는 분포에 적합합니다.
  • 단점
    • 연산 리소스가 많이 소요되고 지연 시간에 민감하여 모델 복잡도가 제한될 수 있습니다.
    • 모니터링 요구사항이 더 많습니다.

오프라인 추론(offline inference): 예측 그룹을 생성하여 저장해 두고 요청에 따라 예측을 검색합니다. 온라인 추론과 대비되는 개념

Reference

https://developers.google.com/machine-learning/crash-course/static-vs-dynamic-inference/video-lecture
