다음 세 클래스는 장고 뷰를 만드는 데 필요한 많은 기능을 제공합니다. 부모보기는 부모보기로 생각할 수 있습니다. 부모보기는 스스로 사용할 수도 있고 상속받을 수도 있습니다. 프로젝트에 필요한 모든 기능을 제공하지 않을 수도 있습니다. 이 경우 Mixins 및 일반 클래스 기반보기가 있습니다.

Django에 내장 된 클래스 기반 뷰의 대부분은 다른 클래스 기반 뷰 또는 다양한 mixins에서 상속받습니다. 이 상속 체인은 매우 중요하기 때문에 조상 클래스는 Ancestors(MRO) 섹션 제목 아래에 문서화되어 있습니다. MRO는 Method Resolution Order의 약자입니다.

View

class django.views.generic.base.View

마스터 클래스 기반의 기본 뷰입니다. 다른 모든 클래스 기반 뷰는이 기본 클래스에서 상속받습니다. 이것은 엄격하게 일반적인 뷰가 아니므로 django.views 에서 가져올 수도 있습니다.

Method Flowchart

  • dispatch()
  • http_method_not_allowed()
  • options()

Example views.py:

from django.http import HttpResponse
from django.views import View

class MyView(View):

    def get(self, request, *args, **kwargs):
        return HttpResponse('Hello, World!')

Example urls.py:

from django.urls import path

from myapp.views import MyView

urlpatterns = [
    path('mine/', MyView.as_view(), name='my-view'),
]

Attributes

  • http_method_names

    이 HTTP 메소드 이름의 목록은 이 view에서 허용 할 것이다.

    Default:

    ['get', 'post', 'put', 'patch', 'delete', 'head', 'options', 'trace']

Methods

  • classmethod as_view(**initkwargs)

    요청을 받아 응답을 반환하는 호출 가능보기를 반환합니다:

    response = MyView.as_view()(request)

    반환된 뷰에는 view_class 및 view_initkwargs 특성이 있습니다.

    요청/응답주기 동안 뷰가 호출되면 HttpRequest 가 뷰의 request 속성에 할당됩니다. URL 패턴에서 캡처 된 위치 및 / 또는 키워드 인수는 각각 args 및 kwargs 특성에 할당됩니다. 그런 다음 dispatch() 가 호출됩니다.

  • dispatch(request, *args, **kwargs)

    view 의 view part - request 인수와 인수를 받아들이고 HTTP 응답을 반환하는 메서드입니다.

    기본 구현은 HTTP 메소드를 검사하고 HTTP 메소드와 일치하는 메소드에 위임을 시도합니다; GET 은 get(), post() 등 POST 로 위임됩니다.

    기본적으로 HEAD request은 get() 에 위임됩니다. GET과 다른 방법으로 HEAD request을 처리해야하는 경우 head() 메서드를 재정의할 수 있습니다.

    예제는 다른 HTTP 메소드 지원(Supporting other HTTP methods)은 참조하십시오.

  • http_method_not_allowed(request, *args, **kwargs)

    view가 지원하지 않는 HTTP 메소드로 호출 된 경우이 메소드가 대신 호출됩니다.

    기본 구현에서는 허용된 메서드 목록을 일반 텍스트로 사용하여 HttpResponseNotAllowed 를 반환합니다.

  • options(request, *args, **kwargs)

    OPTIONS HTTP 동사에 대한 요청에 응답합니다. view의 허용된 HTTP 메서드 이름 목록을 포함하는 Allow header가 포함된 응답을 반환합니다.
