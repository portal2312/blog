각 응용 프로그램에서 제공하는 여러 static files 세트를 다루기가 까다로워 짐.

django.contrib.staticfiles

Static files을 각 Apps 프로그램(및 지정한 다른 위치)에서 프로덕션 환경에서 쉽게 제공 할 수 있는 단일 위치로 수집함.

Customize your app's look and feel

$ mkdir -p ./mysite/polls/static/polls
$ touch ./mysite/polls/static/polls/style.css

Django가 templates(polls/templates/)과 마찬가지로 static files 을 찾음.

Django의 STATICFILES_STORAGE 설정은 다양한 소스에서 정적 파일을 찾는 방법을 알고 있는 finters 목록을 포함.

기본값 중 하나는 AppDirectoriesFinder, 방금 생성 한 설문 조사에서 처럼 INSTALLED_APPS 각각에서 "static" 하위 Dir 찾음. static files에 대해 동일한 Dir 구조 사용.

방금 작성한 static Dir에서 polls 라는 다른 Dir를 작성하고 그 안에 style.css라는 파일을 작성. (polls/static/polls/style.css)

AppDirectoriesFinder static files finder의 작동 방식 때문에 장고의 static files를 간단히 polls/style.css로 참조 할 수 있음. (templates의 경로를 참조하는 방법과 비슷)

* Static file namespacing

templates 마찬가지로
static files을 polls/static (다른 polls 서브 Dir를 작성하지 않고)으로
직접 가져올 수는 있지만 실제로는 좋지 않은 아이디어 임.

Django는 이름이 일치하는 첫 번째 static files을 선택.

다른 응용 프로그램에서 같은 이름의 static files을 가지고 있는 경우
Django는 그들 사이를 구별 하십시오.

우리는 Django가 올바른 것을 가리킬 수 있어야 하며,
이것을 보장하는 가장 쉬운 방법은 namespace를 지정.

즉, static files을 Apps 프로그램 자체의 다른 Dir에 넣으면 됨.

/* ./polls/static/polls/style.css */
li a {
    color: green;
}

<!--polls/templates/polls/index.html-->
{% load static %}

<link rel="stylesheet" type="text/css" href="{% static 'polls/style.css' %}" />

{\% static \%} tamplate tag는 static files의 절대 URL을 생성.

개발하기 위해 반드시 필요.

http://localhost:8000/polls/ reload 그리고 너는 질문 링크가 초록색 (Django style!) 인 것을 볼 수 있음. 이는 스타일 시트가 제대로 load 되었음을 의미함.

Adding a background-image

$ mkdir -p ./polls/static/polls/images
$ touch background.gif

CSS 수정 (내용 추가)

$ vi ./polls/static/polls/style.css

body {
    background: white url("images/bg.png") no-repeat right bottom;
}

Warning:

물론 {\% static \%} template tag는 django가 생성하지 않은 stylesheet와 같은 static files에는 사용할 수 없음.

static files의 경로를 수정하지 않고 STATIC_URL(URL을 생성하기 위해 static template tag에서 사용)을 변경할 수 있기 때문에 항상 상대 경로를 사용하여 static files을 서로 연결해야 함.

이것들은 기본.
