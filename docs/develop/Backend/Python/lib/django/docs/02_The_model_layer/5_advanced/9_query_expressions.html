Built-in Expressions

F() expressions

(작성기준: django-en-1.11)

class F[source]

F() 객체는 모델 필드 또는 주석 된 열의 값을 나타냅니다. 실제로 데이터베이스에서 파이썬 메모리로 가져 오지 않고도 모델 필드 값을 참조하고 이를 사용하여 데이터베이스 작업을 수행 할 수 있습니다.

대신 Django는 F() 객체를 사용하여 데이터베이스 수준에서 필요한 작업을 설명하는 SQL 표현식을 생성.

예:

# Tintin filed a news story!
reporter = Reporters.objects.get(name='Tintin')
reporter.stories_filed += 1
reporter.save()

여기서 우리는 reporter.stories_filed 의 값을 데이터베이스에서 메모리로 가져 와서 친숙한 파이썬 연산자를 사용하여 조작 한 다음 객체를 다시 데이터베이스에 저장했습니다.

그러나 대신 우리는 다음과 같이 할 수 있습니다:

from django.db.models import F

reporter = Reporters.objects.get(name='Tintin')
reporter.stories_filed = F('stories_filed') + 1
reporter.save()

reporter.stories_filed = F('stories_filed') + 1 는 사실 파이썬이 값을 인스턴스 속성에 할당 한 것처럼 보입니다. 실제로는 데이터베이스에서 연산을 설명하는 SQL 구문입니다.

장고는 F ()의 인스턴스를 만나면 표준 파이썬 연산자를 오버라이드하여 캡슐화 된 SQL 표현식을 생성합니다. 이 경우 reporter.stories_filed가 나타내는 데이터베이스 필드를 증가 시키도록 데이터베이스에 지시합니다.

eporter.stories_filed에 어떤 값이 있었는지에 관계없이 파이썬은 그것에 대해 결코 알지 못합니다 - 그것은 데이터베이스에 의해 전적으로 처리됩니다. 모든 파이썬은 Django의 F () 클래스를 통해 필드를 참조하고 작업을 설명하는 SQL 구문을 작성합니다.

이 방법으로 저장된 새 값에 액세스하려면 객체를 다시 로드해야함:

reporter = Reporters.objects.get(pk=reporter.pk)
# Or, more succinctly:
reporter.refresh_from_db()

위와 같이 단일 인스턴스에 대한 작업 에서뿐만 아니라 F()는 update()를 사용하여 객체 인스턴스의 QuerySet에 사용할 수 있습니다. 이것은 위의 get()과 save()를 사용했던 두 개의 쿼리를 하나의 쿼리로 줄입니다:

reporter = Reporters.objects.filter(name='Tintin')
reporter.update(stories_filed=F('stories_filed') + 1)

또한 update ()를 사용하여 여러 객체의 필드 값을 증가시킬 수 있습니다. 데이터베이스에서 파이썬으로 모든 것을 가져 와서 루핑하고, 각각의 필드 값을 증가시키고, 각각을 데이터베이스에 다시 저장하는 것보다 훨씬 빠를 수 있습니다:

Reporter.objects.all().update(stories_filed=F('stories_filed') + 1)

(장점) 따라서 F()는 다음과 같은 방법으로 성능 이점을 제공 할 수 있습니다:

  • Python이 아닌 데이터베이스를 사용하여 작업 수행.

  • 일부 작업에 필요한 쿼리 수 감소.

Avoiding race conditions using F()

F()의 또 다른 유용한 이점은 파이썬이 아닌 데이터베이스가 필드 값을 업데이트하면 경쟁 조건을 피할 수 있다는 것입니다.

위의 첫 번째 예제에서 두 파이썬 스레드가 코드를 실행하면 한 스레드가 다른 스레드가 데이터베이스에서 값을 검색 한 후에 필드 값을 검색, 증가 및 저장할 수 있습니다. 두 번째 스레드가 저장하는 값은 원래 값을 기반으로합니다. 첫 번째 스레드의 작업이 손실됩니다.

데이터베이스에서 필드를 업데이트해야하는 경우 프로세스가보다 강력 해집니다: 인스턴스가 검색 될 때 그 값을 기반으로하는 것이 아니라 save () 또는 update ()가 실행될 때 데이터베이스의 필드 값을 기반으로 필드를 업데이트합니다.

F() assignments persist after Model.save()

모델 필드에 할당 된 F () 객체는 모델 인스턴스를 저장 한 후에도 유지되며 각 save()에 적용됩니다:

reporter = Reporters.objects.get(name='Tintin')
reporter.stories_filed = F('stories_filed') + 1
reporter.save()

reporter.name = 'Tintin Jr.'
reporter.save()

stories_filed는이 경우에 두 번 업데이트됩니다. 처음에 1이면 최종 값은 3이됩니다.

Using F() in filters

F ()는 QuerySet 필터에서 매우 유용합니다. QuerySet 필터를 사용하면 Python 값이 아닌 필드 값을 기준으로 객체 집합을 필터링 할 수 있습니다.

이것은 쿼리에서 F() 표현식을 사용할 때 문서화됩니다. using F() expressions in queries

Using F() with annotations

F ()는 다양한 필드를 산술로 결합하여 모델에 동적 필드를 만드는 데 사용할 수 있습니다.

company = Company.objects.annotate(
    chairs_needed=F('num_employees') - F('num_chairs'))

만약 당신이 결합하고있는 필드가 다른 타입이라면 장고에게 어떤 종류의 필드가 반환 될지를 알려줘야합니다. F ()는 output_field를 직접 지원하지 않으므로 ExpressionWrapper로 표현식을 래핑해야합니다:

from django.db.models import DateTimeField, ExpressionWrapper, F

Ticket.objects.annotate(
    expires=ExpressionWrapper(
        F('active_at') + F('duration'), output_field=DateTimeField()))

ForeignKey와 같은 관계형 필드를 참조 할 때 F ()는 모델 인스턴스가 아닌 기본 키 값을 반환합니다:

>> car = Company.objects.annotate(built_by=F('manufacturer'))[0]
>> car.manufacturer
<Manufacturer: Toyota>
>> car.built_by
3
