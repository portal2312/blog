Managing static files (e.g. images, JavaScript, CSS)

Documentation version: 1.11

웹 사이트는 일반적으로 이미지, 자바 스크립트 또는 CSS와 같은 추가 파일을 제공해야 합니다.

Django에서는 이러한 파일을 "정적 파일"이라고 부릅니다.

Django는 django.contrib.staticfiles를 제공하여이를 관리 할 수 ​​있도록 도와줍니다.

Configuring static files

  1. django.contrib.staticfiles가 INSTALLED_APPS에 포함되어 있는지 확인하십시오.

  2. settings files에서 STATIC_URL을 정의하십시오. 예:

  STATIC_URL = '/static/'
  1. templates에서, /static/my_app/example.jpg 와 같은 URL을 하드 코드하거나, static tamplate tag를 사용하여 구성된 STATICFILES_STORAGE 저장소를 사용하여 지정된 상대 경로에 대한 URL을 작성하는 것이 좋습니다 (원하는 경우 훨씬 쉽게 할 수 있습니다). static files을 제공하기 위한 콘텐츠 전달 네트워크 (CDN)로 전환합니다.
  {% load static %}
  <img src="{% static "my_app/example.jpg" %}" alt="My image"/>
  1. 정적 파일을 앱의 정적 폴더에 저장하십시오. 예: my_app/static/my_app/example.jpg.
  NOTE: Serving the files

  이러한 구성 단계 외에도 실제로 정적 파일을 제공해야합니다.

  개발하는 동안 **django.contrib.staticfiles** 를 사용하면 **DEBUG** 가 **True** 설정된 경  우 **runserver** 가 자동으로 수행. (**django.contrib.staticfiles.views.serve()** 참조)

  이 방법은 매우 비효율적이며 안전하지 않을 수 있으므로 프로덕션에는 적합하지 않습니다.

  프로덕션 환경에서 [정적 파일을 제공][Delpoying static files] 하기 위한 올바른 전략에 대한 정적 파일 배포를 참조하십시오.

프로젝트에는 특정 앱에 묶여 있지 않은 고정 자산도있을 것입니다. apps 내부에 static/ 경로를 사용하는 것 외에도, Django가 정적 파일을 찾는 설정 파일에서 디렉토리 목록 (STATICFILES_DIRS)을 정의 할 수 있습니다: py STATICFILES_DIRS = [ os.path.join(BASE_DIR, "static"), '/var/www/static/', ]

staticfiles 을 찾는 방법에 대한 자세한 내용은 STATICFILES_FINDERS 설정에 대한 설명서를 참조하십시오.

NOTE: Static file namespacing

이제 정적 파일을 my_app/static/ 에 직접 두는 것 (다른 my_app 하위 디렉토리를 만드는 것보다)을 피할 수는 있지만 실제로는 좋지 않습니다. 그러나 그것은 실제로 나쁜 생각 일 것입니다. Django는 이름이 일치하는 첫 번째 정적 파일을 사용합니다. 다른 응용 프로그램에 같은 이름의 정적 파일이 있으면 Django는 이들을 구별하지 못합니다. 우리는 Django가 올바른 것을 가리킬 수 있어야하며, 이것을 보장하는 가장 쉬운 방법은 네임 스페이스를 지정하는 것입니다. 즉, 정적 파일을 응용 프로그램 자체의 다른 디렉토리에 넣으면 됩니다.

Serving static files during development

위에서 설명한대로 django.contrib.staticfiles 를 사용하면 DEBUG가 True로 설정된 경우 runserver 가 자동으로이 작업을 수행합니다. INSTALLED_APPS 에 django.contrib.staticfiles 가 없으면 django.views.static.serve() view를 사용하여 수동으로 static files을 제공할 수 있습니다.

프로덕션 용도로는 적합하지 않습니다! 일반적인 배포 전략 중 일부는 정적 파일 배포를 참조하십시오.

예를 들어 STATIC_URL 이 /static/ 으로 정의된 경우 urls.py에 다음 스니펫을 추가하여이 작업을 수행 할 수 있습니다:

from django.conf import settings
from django.conf.urls.static import static

urlpatterns = [
    # ... the rest of your URLconf goes here ...
] + static(settings.STATIC_URL, document_root=settings.STATIC_ROOT)

NOTE

이 도우미 함수는 디버그 모드에서만 작동하며 지정된 접두어가 URL (예: http://static.example.com/)이 아닌 로컬 (예: /static/)인 경우에만 작동합니다.

또한이 도우미 함수는 실제 STATIC_ROOT 폴더만 제공합니다.
django.contrib.staticfiles와 같은 정적 파일 검색을 수행하지 않습니다.
