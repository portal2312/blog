Prerequisites

Docker EE customers

생략

OS requirements

다음 우분투 버전 중 하나의 64 비트 버전이 필요합니다.

  • Zesty 17.04
  • Xenial 16.04 (LTS) - Long Term Support
  • Trusty 14.04 (LTS)

Docker CE 는 x86_64, armhf 및 s390x (IBM z Systems) 아키텍처의 Ubuntu에서 지원됩니다.

s390x Limitations: System Z 는 Ubuntu Xenial 및 Zesty에서만 지원됩니다.

Uninstall old versions

Docker의 이전 버전은 docker 또는 docker-engine으로 불렸습니다. 설치되어 있으면 제거하십시오.

apt-get remove docker docker-engine docker.io

apt-get 이 설치되어 있지 않다는 보고가 있으면 괜찮습니다.

images, containers, volumns 그리고 networks 포함하여 /var/lib/docker/ 의 내용이 보존됩니다.

Docker CE packages 는 이제 docker-ce 라고 합니다.

Recommended extra packages for Trusty 14.04

생략

Install Docker CE

다양한 방법으로 설치

  • 대부분의 사용자는 설치 및 업그레이드 작업을 쉽게하기 위해 set up Docker's repositories 하고 설치합니다.

이 방법이 권장됩니다.

  • 일부 사용자는 DEB 패키지를 다운로드하여 install it manually(수동 설치) 하고 업그레이드를 완전히 수동으로 관리합니다.

이 기능은 Docker를 인터넷에 접속할 수없는 에어 갭 시스템에 설치하는 경우에 유용합니다.

  • 테스트 및 개발 환경에서 일부 사용자는 자동화 된 convenience scripts 를 사용하여 Docker를 설치하기로 선택합니다.

Install using the repository

새 호스트 시스템에 Docker CE를 처음 설치하기 전에 Docker 저장소를 설정해야합니다.

그런 다음 저장소에서 Docker를 설치하고 업데이트 할 수 있습니다.

SET UP THE REPOSITORY

  1. Update the apt package index:

    $ sudo apt-get update

  2. apt가 HTTPS를 통해 저장소를 사용할 수 있도록 패키지를 설치하십시오.

    $ sudo apt-get install \
    apt-transport-https \
    ca-certificates \
    curl \
    software-properties-common

  3. Docker 공식 GPG key 추가:

    $ sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -

    지문의 마지막 8자를 검색하여 지문 인식 키 9DC8 5822 9FC7 DD38 854A E2D8 8D81 803C 0EBF CD88가 있는지 확인하십시오.

    $ sudo apt-key fingerprint 0EBFCD88

# pub   rsa4096 2017-02-22 [SCEA]
# 9DC8 5822 9FC7 DD38 854A  E2D8 8D81 803C 0EBF CD88
# uid           [ unknown] Docker Release (CE deb) <docker@docker.com>
# sub   rsa4096 2017-02-22 [S]

  4. stable 저장소를 설정하려면 다음 명령을 사용하십시오. edge 또는 test 저장소에서 빌드를 설치하려는 경우에도 항상 stable 저장소가 필요합니다. edge 또는 test 저장소를 추가하려면 아래 명령에서 stable 이라는 단어 뒤에 edge 또는 test (또는 둘 다)를 추가하십시오.

    Note: 아래의 lsb_release -cs 하위 명령은 xenial과 같은 우분투 배포본의 이름을 반환합니다. 때로는 Linux Mint와 같은 배포판에서 $(lsb_release -cs)를 상위 우분투 배포본으로 변경해야 할 수도 있습니다. 예를 들어, Linux Mint Rafaela를 사용하는 경우 trusty할 수 있습니다.

    amd64:

    $ sudo add-apt-repository \
    "deb [arch=amd64] https://download.docker.com/linux/ubuntu \
    $(lsb_release -cs) \
    stable"

    armhf:

    $ sudo add-apt-repository \
    "deb [arch=armhf] https://download.docker.com/linux/ubuntu \
    $(lsb_release -cs) \
    stable"

    s390x:

    $ sudo add-apt-repository \
    "deb [arch=s390x] https://download.docker.com/linux/ubuntu \
    $(lsb_release -cs) \
    stable"

    Note: Docker 17.06부터 안정 버전은 edge 및 test 저장소로 푸시됩니다.

    Learn about stable and edge channels

INSTALL DOCKER CE

  1. Update the apt package index.

    $ sudo apt-get update

  2. Docker CE 최신 버전을 설치하거나 다음 단계로 이동하여 특정 버전을 설치하십시오.

Docker의 기존 설치가 모두 대체됩니다.

```bash
$ sudo apt-get install docker-ce
```

> Q. 여러 개의 Docker 저장소가 있습니까?
>
> Docker repository 가 여러 개있는 경우 `apt-get install` 또는 `apt-get update` 명령에서 버전을 지정하지 않고 설치 또는 업데이트하면 항상 가능한 가장 높은 버전이 설치되므로 안정성 요구에 적합하지 않을 수 있습니다.

  1. 프로덕션 시스템에서는 항상 최신 버전을 사용하는 대신 특정 버전의 Docker CE를 설치해야 합니다.

    MY-Note: 해당 3 은 2와 다른 설치 방법을 설명한다. (2로 설치시 넘어가기)

    $ apt-cache madison docker-ce | 17.09.0~ce-0~ubuntu | https://download.docker.com/linux/ubuntu xenial/stable amd64 Packages

    목록의 내용은 사용할 수있는 리포지토리에 따라 다릅니다.

    설치할 특정 버전을 선택하십시오.

    두 번째 열은 버전 문자열입니다.

    세 번째 열은 저장소 이름이며, 패키지가있는 저장소와 안정성 수준을 나타냅니다.

    특정 버전을 설치하려면 패키지 이름에 버전 문자열을 추가하고 등호 (=)로 구분하십시오:

    $ sudo apt-get install docker-ce=<VERSION>

    Docker daemon 는 자동으로 시작됩니다.

  2. hello-world 이미지를 실행하여 Docker CE가 올바르게 설치되었는지 확인하십시오.

    $ sudo docker run hello-world

# Unable to find image 'hello-world:latest' locally
# latest: Pulling from library/hello-world
# 5b0f327be733: Pull complete
# Digest: sha256:07d5f7800dfe37b8c2196c7b1c524c33808ce2e0f74e7aa00e603295ca9a0972
# Status: Downloaded newer image for hello-world:latest
#
# Hello from Docker!
# This message shows that your installation appears to be working correctly.
#
# To generate this message, Docker took the following steps:
# 1. The Docker client contacted the Docker daemon.
# 2. The Docker daemon pulled the "hello-world" image from the Docker Hub.
# 3. The Docker daemon created a new container from that image which runs the
# executable that produces the output you are currently reading.
# 4. The Docker daemon streamed that output to the Docker client, which sent it
# to your terminal.
#
# To try something more ambitious, you can run an Ubuntu container with:
# $ docker run -it ubuntu bash
#
# Share images, automate workflows, and more with a free Docker ID:
# https://cloud.docker.com/
#
# For more examples and ideas, visit:
# https://docs.docker.com/engine/userguide/

    이 명령은 테스트 이미지를 다운로드하여 컨테이너에서 실행합니다.

    컨테이너가 실행되면 정보 메시지를 인쇄하고 종료합니다.

Docker CE가 설치되어 실행 중입니다.

Docker 명령을 실행하려면 sudo 를 사용해야 합니다.

권한이 없는 사용자가 Docker 명령을 실행하고 다른 선택적 구성 단계를 수행 할 수 있도록 Linux postinstall 를 계속하십시오.

Install from a package

Docker의 저장소를 사용하여 Docker CE를 설치할 수 없는 경우 릴리스 용 .deb 파일을 다운로드하여 수동으로 설치할 수 있습니다.

Docker CE를 업그레이드 할 때마다 새 파일을 다운로드해야 합니다.

  1. https://download.docker.com/linux/ubuntu/dists/에서 우분투 버전을 선택하고

pool/stable/로 이동 한 다음 amd64, armhf 또는 s390x 를 선택하십시오.

설치할 Docker 버전의 .deb 파일을 다운로드하십시오.

Note: edge package를 설치하려면 URL의 stable 이라는 단어를 edge로 변경하십시오.

Learn about stable and edge channels에 대해 알아보십시오.

  1. Docker CE 설치하고, 아래 경로를 Docker package 다운로드한 경로 변경하기

    $ sudo dpkg -i /path/to/package.deb

    Docker daemon 은 자동으로 시작한다.

  2. hello-world 이미지를 실행하여 Docker CE가 올바르게 설치되었는지 확인하십시오.

    $ sudo docker run hello-world

Docker CE 가 설치되어 실행 중입니다.

Docker 명령을 실행하려면 sudo를 사용해야합니다.

비 특권 사용자가 Docker 명령을 실행하고 다른 선택적 구성 단계를 수행 할 수 있도록 Linux postinstall 를 계속하십시오.

UPGARDE DOCKER CE

최신 패키지 파일을 다운로드하고 설치 절차를 반복하여 새 파일을 가리킵니다.

UPGRADE DOCKER AFTER USING THE CONVENIENCE SCRIPT

Uninstall Docker CE

  1. 패키지 삭제

    $ sudo apt-get purge docker-ce

  2. host 의 images, containers, volumns 또는 customized 구성 파일은 자동으로 제거되지 않습니다. 모든 것을 삭제하려면:

    $ sudo rm -rf /var/lib/docker

편집된 구성 파일은 수동으로 삭제해야 합니다.

Install from a package

Docker의 저장소를 사용하여 Docker CE를 설치할 수 없는 경우 릴리스 용 .deb 파일을 다운로드하여 수동으로 설치할 수 있습니다.

Docker CE를 업그레이드 할 때마다 새 파일을 다운로드해야 합니다.

  1. https://download.docker.com/linux/ubuntu/dists/에서 우분투 버전을 선택하고

pool/stable/로 이동 한 다음 amd64, armhf 또는 s390x 를 선택하십시오.

설치할 Docker 버전의 .deb 파일을 다운로드하십시오.

Note: edge package를 설치하려면 URL의 stable 이라는 단어를 edge로 변경하십시오.

Learn about stable and edge channels에 대해 알아보십시오.

  1. Docker CE 설치하고, 아래 경로를 Docker package 다운로드한 경로 변경하기

    $ sudo dpkg -i /path/to/package.deb

    Docker daemon 은 자동으로 시작한다.

  2. hello-world 이미지를 실행하여 Docker CE가 올바르게 설치되었는지 확인하십시오.

    $ sudo docker run hello-world

Docker CE 가 설치되어 실행 중입니다.

Docker 명령을 실행하려면 sudo를 사용해야합니다.

비 특권 사용자가 Docker 명령을 실행하고 다른 선택적 구성 단계를 수행 할 수 있도록 Linux postinstall 를 계속하십시오.

Uninstall Docker CE

  1. 패키지 삭제

    $ sudo apt-get purge docker-ce

  2. host 의 images, containers, volumns 또는 customized 구성 파일은 자동으로 제거되지 않습니다. 모든 것을 삭제하려면:

    $ sudo rm -rf /var/lib/docker

편집된 구성 파일은 수동으로 삭제해야 합니다.
