이미지 관리

조직 내부 또는 외부의 다른 사람들이 이미지를 사용할 수있게하는 가장 쉬운 방법은 - Docker Hub - Docker Trusted Registry(#content-trust) 와 같은 Docker Registry - private Registry(#docker-trusted-registry)를 실행하는 것입니다.

Docker Hub

Docker Hub는 Docker, Inc.에서 관리하는 공용 레지스트리입니다.

조직, 사용자 계정 및 이미지에 대한 정보를 중앙 집중식으로 관리합니다.

여기에는 docker login, docker pull, docker push, 주석, stars, 검색 등과 같은 명령을 사용하여 조직, CLI 및 API 액세스를 사용하는 웹 UI, 인증 및 권한 부여가 포함됩니다.

Docker Hub는 또한 [Docker Store]에 통합되어 있습니다.

[Docker Store]는 무료가 아닌 이미지에 대한 인 타이틀먼트를 구매 및 판매 할 수있는 마켓 플레이스입니다.

Docker Registry

Docker Registry는 Docker의 생태계 구성 요소입니다.

레지스트리는 이름이 붙은 Docker 이미지를 보유하고 있는 저장 및 컨텐트 전달 시스템으로 다양한 태그 버전으로 제공됩니다.

예를 들어 이미지 distribution/registry는 2.0 및 latest가 있습니다.

사용자는 docker pull myregistry.com/stevvooe/batman:voice와 같은 docker push 및 pull 명령을 사용하여 레지스트리와 상호 작용합니다.

Docker Hub는 Docker Registry의 인스턴스입니다.

Docker Trusted Registry

Docker Trusted Registry는 Docker Enterprise Edition의 일부로, 이미지 서명 및 콘텐츠 신뢰, 역할 기반 액세스 제어 및 기타 Enterprise 급 기능과 같은 기능을 포함하는 개인 보안 Docker 레지스트리입니다.

Content Trust

네트워크로 연결된 시스템간에 데이터를 전송할 때 신뢰가 가장 큰 관심사입니다.

특히 인터넷과 같은 신뢰할 수없는 매체를 통해 통신 할 때는 시스템이 작동하는 모든 데이터의 무결성 및 게시자를 보장하는 것이 중요합니다.

Docker를 사용하여 이미지(데이터)를 레지스트리로 밀고 당깁니다.

콘텐츠 신뢰를 통해 모든 채널에서 레지스트리로부터받은 모든 데이터의 무결성 및 게시자를 확인할 수 있습니다.

Docker 클라이언트에서이 기능을 구성하고 사용하는 방법은 Content Trust를 참조하십시오.

Reference

https://docs.docker.com/develop/develop-images/image_management/#docker-hub
