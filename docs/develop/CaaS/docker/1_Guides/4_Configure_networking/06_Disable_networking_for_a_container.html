Disable networking for a container

Table of contents

  • Reference

컨테이너에서 네트워킹 스택을 완전히 비활성화하려면 컨테이너를 시작할 때 --network none flag를 사용할 수 있습니다.

컨테이너 내에서 루프백 장치만 생성됩니다. 다음 예는 이것을 보여줍니다.

  1. container 생성

    $ docker run --rm -dit \
  --network none \
  --name no-net-alpine \
  alpine:latest \
  ash

  2. 컨테이너 내에서 몇 가지 일반적인 네트워킹 명령을 실행하여 컨테이너의 네트워크 스택을 확인하십시오. eth0가 생성되지 않았습니다.

    $ docker exec no-net-alpine ip link show

1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN qlen 1
    link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00
2: tunl0@NONE: <NOARP> mtu 1480 qdisc noop state DOWN qlen 1
    link/ipip 0.0.0.0 brd 0.0.0.0
3: ip6tnl0@NONE: <NOARP> mtu 1452 qdisc noop state DOWN qlen 1
    link/tunnel6 00:00:00:00:00:00:00:00:00:00:00:00:00:00:00:00 brd 00:00:00:00:00:00:00:00:00:00:00:00:00:00:00:00

    $ docker exec no-net-alpine ip route

    두 번째 명령은 라우팅 테이블이 없으므로 empty를 반환합니다.

  3. 컨테이너를 중지하십시오. --rm 플래그로 작성 되었기 때문에 자동으로 제거됩니다.

    $ docker container rm no-net-alpine

Reference

https://docs.docker.com/network/none/
