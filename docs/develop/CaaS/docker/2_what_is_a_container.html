Package software into standardized units for development, shipment and deployment

컨테이너 이미지는 코드, 런타임, 시스템 도구, 시스템 라이브러리, 설정 등 소프트웨어를 실행하는 데, 필요한 모든 것을 포함하는 경량의 독립 실행형 실행 패키지입니다.

Linux 및 Windows 기반 응용 프로그램 모두에서 사용할 수 있는 컨테이너화 소프트웨어는 환경에 관계없이 항상 동일하게 실행됩니다.

컨테이너는 개발 환경과 준비 환경의 차이점과 같은 주변 환경으로부터 소프트웨어를 격리하고 동일한 인프라에서 다른 소프트웨어를 실행하는 팀 간의 충돌을 줄이는데 도움이 됩니다.

LIGHTWEIGHT

단일 컴퓨터에서 실행되는 Docker 컨테이너는 해당 컴퓨터의 운영 체제 커널을 공유합니다.

그들은 즉시 시작하고 덜 계산 및 RAM을 사용합니다.

이미지는 파일 시스템 레이어로 구성되며 공통 파일을 공유합니다.

이렇게하면 디스크 사용이 최소화되고 이미지 다운로드가 훨씬 빨라집니다.

STANDARD

Docker 컨테이너는 개방형 표준을 기반으로하며 모든 주요 Linux 배포판, Microsoft Windows 및 VM, 베어 메탈 및 클라우드를 포함한 모든 인프라에서 실행됩니다.

SECURE

Docker 컨테이너는 서로 및 기본 인프라에서 응용 프로그램을 격리합니다.

Docker는 응용 프로그램 문제를 전체 시스템이 아닌 단일 컨테이너로 제한하기 위해 가장 강력한 기본 격리를 제공합니다.

Comparing Containers and Virtual Machines

컨테이너와 가상 컴퓨터는 리소스 격리 및 할당 이점이 비슷하지만, 컨테이너가 하드웨어가 아닌 운영 체제를 가상화하기 때문에 기능이 다르게 작동합니다.

컨테이너는 더 휴대 가능하고 효율적입니다.

Containers

컨테이너는 코드와 종속성을 함께 패키징하는 앱 계층의 추상화입니다.

여러 컨테이너를 동일한 시스템에서 실행하고 OS 커널을 다른 컨테이너와 공유 할 수 있습니다.

각 컨테이너는 사용자 공간에서 분리 된 프로세스로 실행됩니다.

컨테이너는 VM보다 공간을 적게 차지하며 (컨테이너 이미지는 일반적으로 수십 MB 크기 임) 거의 즉시 시작됩니다.

Virtual Machines

가상 머신 (VM)은 하나의 서버를 여러 서버로 전환시키는 물리적 하드웨어의 추상화입니다.

하이퍼 바이저를 사용하면 여러 대의 VM을 단일 시스템에서 실행할 수 있습니다.

각 VM에는 운영 체제의 전체 복사본, 하나 이상의 응용 프로그램, 필요한 바이너리 및 라이브러리가 포함되어 있습니다 (수십 GB를 차지함).

VM도 부팅 속도가 느릴 수 있습니다.

Containers and Virtual Machines Together

함께 사용되는 컨테이너와 VM은 응용 프로그램을 배포하고 관리하는 데있어 많은 유연성을 제공합니다.

이미지 생략.

Container Standards and Industry Leadership

2013년에 Docker가 출시됨에 따라 소프트웨어 컨테이너를 대중에게 가져옴으로써 앱 개발에 혁명이 시작되었습니다. 불과 몇 년 만에 Docker는 앱 개발, 패키징 및 관리에 새로운 어휘, 프레임 워크 및 표준으로 업계를 변화 시켰습니다.

dotCloud의 피벗으로 시작하여 dotCloud를 더 나은 Linux 컨테이너 기술로 전환 할 수 있습니다. 이 기술은 이식성 있고 유연하며 배포하기 쉽습니다. Docker는 libcontainer를 오픈 소스로 사용하고 전 세계 커뮤니티와 공동으로 개발을 진행했습니다. 단 몇 년 후인 2015년 6월 Docker는 OCI (Open Container Initiative)에 runC 라고 하는 사양 및 런타임 코드를 기증하여 컨테이너 생태계가 성장하고 성숙 해짐에 따라 표준화를 수립했습니다.

이 진화에 따라 Docker는 계속해서 컨테이너 프로젝트를 포기합니다. Containerd는 Docker 엔진 데몬의 핵심 컨테이너 런타임으로 단순성, 견고성 및 이식성에 중점을 둔 산업 표준 컨테이너 런타임으로 상위 수준 시스템에 내장 할 수있는 구성 요소로 설계되었습니다. 도커 엔진은 runC 및 containerd를 기반으로합니다. 이 공동 작업 프로젝트는 개발자를 위한 이식성, OCI를 통한 컨테이너 표준화, 성장하는 도구의 생태계 및 공동체가 공유하고 구축할 수 있는 사전 제작 된 이미지의 무료 시장을 가능하게 합니다.

containerd 는 단순성, 견고 함 및 휴대성을 강조한 업계 표준 컨테이너 런타임입니다. Linux 및 Windows 용 데몬으로 사용할 수 있으며 호스트 시스템의 전체 컨테이너 수명주기( 이미지 전송 및 저장, 컨테이너 실행 및 감독, 저수준 스토리지 및 네트워크 첨부)를 관리할 수 ​​있습니다.
