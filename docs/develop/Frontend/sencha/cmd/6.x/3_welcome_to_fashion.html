What is Fashion?

CSS를 생성하기 위해 사용하는 .scss 파일의 compiler.

테마 또는 어플리케이션에 정의된 값들을 편집하거나 가시적으로 검토시 사용하는

Sencha Inspector 와 같은 툴을 지원하는 일부 추가된 기능도 있음.

Compatibility (호환성)

CSS3 문법과 호환.

나중에 언급할 추가적인 기능들로 인해 Fashion이 Sass의 단순한 구현에 불과하다거나 Fashion이 Sass를 컴파일 한다고만 할 수는 없다. "sass"라는 단어는 폴더 이름 또는 설정 옵션의 이름으로 아직 많은 곳에서 사용되고 있다. 호환성을 위해서 이러헌 설정 옵션들은 엄격하게는 "Sass"와 관련이 없지만 여전히 "sass"라는 이름을 사용.

Using Live Updates

브라우저 application 열기 > Sass 파일 생성된 CSS 파일 대신 로딩됨

Fashion 파일 수정에 대응하여 page를 다시 읽지 않고서도 Sass를 다시 compile 하고 css 업데이트.

sencha app watch와 함께 Fashion을 사용하는 두 가지 방법:

  • app.json 파일의 development 객체를 편집:
  // app.json
  "development" : {
      // ...
      "tags" : [
          "fashion"
      ]
      // ...
  },
  • "?platformTags=fashion:1"를 로딩하려고 하는 페이지 URL에 추가하는 것.
$ sencha app watch

브라우저에서 해당 어플리케이션에 접속 (http://localhost:1841/app)

이제 테마 변수들을 편집하고 수정 후 저장하면, 변경사항이 즉시 브라우저에 로딩된 어플리케이션에 반영되는 것을 확인할 수 있을 것.

주의: 라이브 업데이트는 Cmd 웹 서버를 사용해서 브라우저로 로딩할 경우에만 동작.

ExtJS Toolkit

  • classic 일부 Sass 변경은 전체 페이지의 리로딩이 불가피.

  • modern 거의 없음.

이는 CSS3에 의존도가 더 높으며, 적극적으로 변경사항을 수용했기 때문.

Language Extensions

Dynamic Variables

  • Fashion에서 매우 중요한 역할을 담당.

  • 변수 값이 dynamic() 안에 들어가 있음.

  • 서로 상호 작용함.

  • 할당(assignment) 및 평가(evaluation) 두 단계를 거쳐 처리

  $bar : dynamic(blue);

  $foo : dynamic($bar); // $foo는 $bar에 의해 결정 됨

  $bar : dynamic(red);

  @debug $foo; // red

이러한 의존성은 Fashion에서 특별 취급되어 foo 값은 bar의 최종 값이 알려질 때까지 연산을 보류함.

assignment (할당)

cascade 순서로 할당(일반변수와 같음)

$bar : dynamic(blue);

$bar : dynamic(red);

@debug $bar; //red

따라서 코드를 통해 변수 값을 지정하고 통제할 수 있게 됨.

동적 변수에 대한 할당은 제어 구조를 적용할 수 없으며, 파일 scope에서만 가능.

잘못된 사용의 경우:

$bar : dynamic(blue);

@if something {
    $bar : dynamic(red); // 잘못된 예
}

옳은 경우:

$bar : dynamic(if(something, red, blue));

동적 변수는 선언 이후, dynamic() 래퍼를 사용해서 또는 사용하지 않고도 할당 가능.

$bar : dynamic(blue);
$bar : red;             // $bar에 red를 할당
$bar : green !default;  // $bar를 green으로 재할당
$debug $bar;            // green

"default dynamic" 같은 형식은 존재하지 않음.

evaluation (평가)

선언순서가 아닌 "의존 순서(dependency order)"에 의해 평가.

마지막 정의 기준으로 동작.

$bar : dynamic(mix($colorA, $colorB, 20%));
$bar : dynamic(lighten($colorC, 20%));  // XXX: 마지막 정의로 결정

Hoisting (견인)

위의 모든 것을 실현하기 위해서

Sass코드의 실행 전, 모든 동적 변수들을 취합하고 평가.

다른 말로, javascript 변수와 유사하게, 동적 변수들은 top으로 견인됨.

Elevation (승격)

변수가 동적 변수에 할당되면, 동적 변수로 승격됨.

$foo : blue;  // XXX: 일반 변수 --> $bar에 의해 동적 변수
$bar : dynamic($foo);  // XXX: 동적 변수 ->> $foo 동적 변수 승격

주의: 승격된 변수는 승격된 규칙을 따라야 함.

이러한 속성은 과거 버전의 Sencha Cmd로부터 이식성을 최대화 하기 위해 지원된다.

변수가 승격될 때, 경고가 콘솔에 생성되는데 향후 릴리즈에서 이 경고는 에러로 변경될 예정.

콘솔에 출력되는 경고를 참고하여 필요한 변수를 dynamic()을 사용해서 수정할 것을 권고.

Extentions - Requiring Javascript

javascript 사용해서 Fashion을 확장 가능.

Sass 코드에서 불러오기 : require()

require('my-module');

// or

require('../*.js')  // XXX: sass 파일 기준 상대 경로 지정

ES6 지원.

Upgrading

Dynamic Variables

당장 필요하지 않더라도 변수 할당을 dynamic()을 사용하도록 수정할 것을 권고.

대부분의 경우에 이러한 수정은 !default를 dynamic()으로 단순 교체하면 문제가 없을 것.

// before:
$base-color : green !default;

// after
$base-color : dynamic(green);

Compass Extensions

더 이상 존재하지 않음.
