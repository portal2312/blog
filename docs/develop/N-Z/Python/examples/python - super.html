How to use

class 다중 상속시 문제:

class A:
   def __init__(self):
      print("Class A __init__()")


class B(A):
   def __init__(self):
      print("Class B __init__()")
      A.__init__(self)


class C(A):
   def __init__(self):
      print("Class C __init__()")
      A.__init__(self)


class D(B, C):
   def __init__(self):
      print("Class D __init__()")
      B.__init__(self)
      C.__init__(self)


d = D()

# Class D __init__()
# Class B __init__()
# Class A __init__()
# Class C __init__()
# Class A __init__()  # iss, 중복

Solution:

class A:
   def __init__(self):
      print("Class A __init__()")
      super(A, self).__init__()


class B(A):
   def __init__(self):
      print("Class B __init__()")
      super(B, self).__init__()


class C(A):
   def __init__(self):
      print("Class C __init__()")
      super(C, self).__init__()


class D(B, C):
   def __init__(self):
      print("Class D __init__()")
      super(D, self).__init__()


d = D()

# Class D __init__()
# Class B __init__()
# Class C __init__()
# Class A __init__()  # 최상단 부모 class 한번만 호출하기

즉, class를 사용하는 입장에서 전체 클래스의 상속 구조를 이해하지 못하였어도, 문제 없이 상속을 사용할 수 있다.

super() 함수는 슈퍼클래스의 method를 호출하라는 의미인데, 이때 다수의 슈퍼 클래스가 존재시 클래스 호출 순서의 결정은 mro 를 통해 결정된다.

mro (Method Resolution Order)

(<class '__main__.D'>, <class '__main__.B'>, <class '__main__.C'>, <class '__main__.A'>, <type 'object'>)

위와 같이 순서대로 호출되게 된다.

Python-2.x

supper()

Note: super() only works for new-style classes.

new-style-class

기본적으로 new-style 이 아닌 old-style class로 생성된다.

old-style class는 형태가 type이 아닌 classobj 이기 때문에, super 함수에서 old-style class 로 할 경우 TypeError가 발생한다.

이러한 문제를 피하기 위해서는 반드시 class 정의시 아래와 같이 명시하여 주어야 한다:

class A(object):
# ...
