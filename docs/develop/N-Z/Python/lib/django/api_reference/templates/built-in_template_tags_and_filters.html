extends

이 템플릿이 상위 템플릿을 확장한다는 신호입니다.

이 태그는 두 가지 방법으로 사용할 수 있습니다.

  • {\% extends "base.html"\%} (따옴표 포함)은 리터럴 값 "base.html"을 확장 할 상위 템플릿의 이름으로 사용합니다.

  • {\% extends variable \%}는 variable의 값을 사용합니다. 변수가 문자열로 평가되면 Django는 그 문자열을 부모 템플릿의 이름으로 사용합니다. 변수가 Template 객체로 평가되면 Django는 그 객체를 부모 템플릿으로 사용합니다.

자세한 내용 참조 Template inheritance.

문자열 인수는 ./ 또는 ../로 시작하는 상대 경로 일 수 있습니다. 예를 들어, 다음 디렉토리 구조를 가정합니다:

dir1/
    template.html
    base2.html
    my/
        base3.html
base1.html

template.html에서 다음 경로가 유효합니다:

{% extends "./base2.html" %}
{% extends "../base1.html" %}
{% extends "./my/base3.html" %}

NOTE:
New in Django 1.10:

상대 경로를 사용할 수있는 기능이 추가되었습니다.

include

템플릿을로드하고 현재 컨텍스트로 렌더링합니다.

이것은 템플릿 내에 다른 템플릿을 "포함"하는 방법입니다.

템플릿 이름은 작은 따옴표 나 큰 따옴표로 변수 또는 하드 코딩 된 (인용 된) 문자열 일 수 있습니다.

이 예제에는 "foo/bar.html" template의 내용이 포함되어 있습니다:

{% include "foo/bar.html" %}

문자열 인수는 확장 태그에 설명 된대로 ./ 또는 ../로 시작하는 상대 경로 일 수 있습니다.

NOTE: New in Django 1.10:

상대 경로를 사용하는 기능이 추가됨.

포함된 템플릿은 템플릿을 포함하는 템플릿 컨텍스트 내에서 렌더링됩니다:

{% include template_name %}

변수는 context를 받아들이는 render() method가 있는 객체일 수도 있습니다. 이렇게하면 context에서 컴파일된 Template 을 참조 할 수 있습니다.

이 예제에서는 "Hello, John!"출력을 생성합니다:

  • Context: 변수 person 이 "John" 으로 설정되고 변수 인사말이 "Hello"로 설정.

  • Template:

  {% include "name_snippet.html" %}
  • name_snippet.html template:
  {{ greeting }}, {{ person|default:"friend" }}!

키워드 인수를 사용하여 템플릿에 추가 컨텍스트를 전달할 수 있습니다.

{% include "name_snippet.html" with person="Jane" greeting="Hello" %}

제공된 변수만 사용하여 context를 렌더링하려면 (또는 변수가 전혀없는 경우) 유일한 옵션을 사용하십시오. 포함된 템플릿에는 다른 변수를 사용할 수 없습니다:

{% include "name_snippet.html" with greeting="Hi" only %}

포함된 템플릿이 렌더링되는 동안 예외가 발생하면 (누락되거나 구문 오류가있는 경우 포함) template engine의 debug 옵션에 따라 동작이 달라집니다 (설정되지 않은 경우이 옵션의 기본값은 DEBUG 임). 디버그 모드가 켜져 있으면 TemplateDoesNotExist 또는 TemplateSyntaxError 와 같은 예외가 발생합니다. DEBUG 모드가 꺼져 있으면 {\% include \%} 는 포함된 템플릿을 렌더링하는 동안 발생하는 예외를 제외하고 빈 문자열을 반환하는 경고를 django.template logger에 기록합니다.

NOTE: 버전 1.11부터는 더 이상 사용되지 않습니다.

{\% include \%} template tag를 렌더링하는 동안 발생하는 사일런스 예외는 더 이상 사용안함.
Django 2.1에서는 예외가 발생합니다.

NOTE:

include 태그는 "이 하위 템플릿을 렌더링하고 HTML을 포함"의 구현으로 간주되어야하며 "이 하위 템플릿을 구문 분석하고 해당 하위 템플릿의 내용을 마치 부모의 일부인 것처럼 포함"하지 말아야합니다.
즉, 포함 된 템플릿간에 공유 상태가 없음을 의미합니다. 각 포함은 완전히 독립적 인 렌더링 프로세스입니다.

블록은 포함되기 전에 평가됩니다. 즉, 다른 블록의 블록을 포함하는 템플릿에는 이미 평가되고 렌더링 된 블록이 포함됩니다. 예를 들어 확장 템플릿에 의해 재정의 될 수있는 블록이 아닙니다.
