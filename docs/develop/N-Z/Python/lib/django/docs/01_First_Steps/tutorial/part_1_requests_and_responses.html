Creating a Project

$ django-admin startproject mysite
# django-admin startproject <project-name>

mysite/
    manage.py
    mysite/
        __init__.py
        settings.py
        urls.py
        wsgi.py

  • namespace 충돌 고려 (<project-name>)

  • project 경로 설정에 대한 참조글 (보안) ``` - 배경이 평범한 구형 PHP (현대 프레임 워크를 사용하지 않음) 인 경우 웹 서버의 문서 루트 (/var/www와 같은 위치) 아래에 코드를 넣는 데 익숙. 장고와 함께, 당신은 그렇게하지 않습니다.

  • 사람들이 웹에서 코드를 볼 수있는 가능성이 있기 때문에이 파이썬 코드를 웹 서버의 문서 루트 내에 두는 것은 좋은 생각이 아닙니다. (보안에 좋지 않음)

  • /home/mycode 와 같은 문서 루트 외부의 일부 디렉토리에 코드를 넣으십시오. ```

  • manage.py : 이 장고 프로젝트와 다양한 방식으로 상호 작용할 수있는 명령 행 유틸리티. django-admin과 manage.py에서 manage.py에 대한 모든 세부 사항을 읽을 수 있습니다.

  • mysite/mysite/ : 내부 mysite Dir은 프로젝트의 실제 파이썬 패키지. 그 이름은 그 안에 무엇이든 가져 오기 위해 사용해야하는 Python 패키지 이름. (ex: mysite.urls)

  • settings.py : 장고 프로젝트의 설정. Django settings

  • urls.py : URL 선언. URL dispctcher

  • wsgi.py : Project 가 wsgi 호환하는 웹 서버 경우 사용. How to deploy with WSGI

  • Changing the Port
$ python manage.py runserver 8000          # XXX: Port
$ python manage.py runserver 0.0.0.0:8000  # XXX: URL:Port

# 127.0.0.1 - localAddress 명시

  • Automatic reloading of runserver
- 개발 서버는 필요에 따라 각 요청에 대한 Python 코드를 자동으로 reload 함.

- 코드 변경 적용: Auto (서버를 다시 시작할 필요가 없음)

- 파일 추가와 같은 일부 작업: Restart. (다시 시작을 trigger하지 않음)

Creating the Polls app

  • Django로 작성하는 각 애플리케이션은 특정 규칙을 따르는 Python 패키지로 구성.

  • Django에는 응용 프로그램의 기본 디렉토리 구조를 자동으로 생성하는 유틸리티가 있음.

  • 디렉토리를 만드는 대신 코드를 작성하는 데 집중할 수 있음.

* Projects vs apps

- apps
  - 웹 로그 시스템, 공개 기록 데이터베이스 또는 간단한 설문 조사 앱과 같은 기능을
    수행하는 웹 애플리케이션입니다.
  - 여러 Projects에 있을 수 있습니다.

- Projects
  - 특정 웹 사이트의 구성 및 응용 프로그램 모음입니다.
  - 여러 개의 apps이 포함될 수 있습니다.

# python manage.py startapp <apps-name>
$ python manage.py startapp polls

polls/
    __init__.py
    admin.py
    apps.py
    migrations/
        __init__.py
    models.py
    tests.py
    views.py

Write your first view

$ vi ./polls/views.py

from django.http import HttpResponse


def index(request):
    content = 'Hello, world.'
    return HttpResponse(content)

  • 뷰를 호출하려면 URL에 매핑해야 함.

  • 이 경우 URLconfs 필요.

  • URLconfs를 만들려면 해당경로 안의 urls.py 만듬. (../polls/urls.py)

$ touch urls.py  # XXX: create - urls.py

polls/
    __init__.py
    admin.py
    apps.py
    migrations/
        __init__.py
    models.py
    tests.py
    urls.py    # XXX: here
    views.py

$ vi ./polls/urls.py

from django.conf.urls import url
from . import views


urlpatterns = [
    url(r'^$', views.index, name='index'),
]

$ vi ../mysite/urls.py

from django.conf.urls import include, url
from django.contrib import admin


urlpatterns = [
    url(r'^admin/', admin.site.urls),
    url(r'^polls/', include('polls.urls')),
]

  • include()

    • from django.conf.urls import include
    • 다른 URLconfs 를 참조 할 수 있게 함.
    • include() 함수의 일반 표현식에는 $(문자열 끝 일치 문자)가 없지만 뒤에 /가 있음에 유의하십시오.
    • Django가 include() 를 발견 할 때마다, 그 점까지 일치하는 URL 부분을 잘라 버리고 추가 처리를 위해 포함된 URLconfs에 나머지 문자열을 보냅니다.
    • include() 뒤에있는 아이디어는 URL을 플러그 앤 플레이하기 쉽도록 만드는 것입니다.
    • polls는 자신의 URLconfs (polls/urls.py)에 있기 때문에 "/polls/" or "/fun_polls/" or "/content/polls/" or 다른 경로 루트 아래에 배치 할 수 있음. 그러면 앱은 여전히 ​​작동.
* When to use in include()

- include()를 사용하는 경우

  다른 URL 패턴을 포함 할 때는 항상 include()를 사용해야 함.
  이것에 대한 유일한 예외는 admin.site.urls 임.

* Doesn't match what you see?

  admin.site.urls 대신 include (admin.site.urls)가 표시되면 django version check.

$ python manage.py runserver

# go to http://localhost:8000/polls/

url() arguments: regex

  • 정규식 사용.

  • 정규식은 GET 과 POST 를 읽지 않음.

https://www.example.com/myapp/?page=3 는 myapp/ URLconf 를 바라봄

url() arguments: view

  • Django가 정규식 일치를 발견하면 Django는 HttpRequest 객체를 첫 번째 인수로 사용하고 정규 표현식의 "캡처 된"값을 다른 인수로 사용하여 지정된보기 함수를 호출합니다.

  • regex가 단순 캡처를 사용하면 값은 위치 인수로 전달됩니다. 명명 된 캡처를 사용하는 경우 값은 키워드 인수로 전달됩니다. 우리는 이것을 약간의 예를 들어 설명 할 것입니다.

url() arguments: kwargs

  • 임의의 키워드 인수는 사전에서 대상보기로 전달 될 수 있습니다. 튜토리얼에서 장고의 이 기능을 사용하지 않을 것입니다.

url() arguments: name

  • URL의 이름을 지정하면 특히 템플릿 내에서 장고의 다른 곳에서 명확하게 참조 할 수 있습니다. 이 강력한 기능을 사용하면 단일 파일 만 건드리지 만 프로젝트의 URL 패턴을 전체적으로 변경할 수 있습니다.
