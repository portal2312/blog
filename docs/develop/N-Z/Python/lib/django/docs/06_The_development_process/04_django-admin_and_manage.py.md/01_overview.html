dumpdata

django-admin dumpdata [app_label[.ModelName] [app_label[.ModelName] ...]]

명명 된 응용 프로그램과 연관된 데이터베이스의 모든 데이터를 표준 출력으로 출력합니다.

응용 프로그램 이름이 제공되지 않으면 설치된 모든 응용 프로그램이 덤프됩니다.

dumpdata의 출력은 loaddata의 입력으로 사용될 수 있습니다.

dumpdata는 덤프 할 레코드를 선택하기 위해 모델의 기본 관리자를 사용합니다. 사용자 정의 관리자를 기본 관리자로 사용하고 사용 가능한 일부 레코드를 필터링하는 경우 모든 오브젝트가 덤프되지는 않습니다.

djang-admin dumpdata 시 virtualenv 환경인 경우 Error 발생:

django.core.exceptions.ImproperlyConfigured:
  Requested setting INSTALLED_APPS, but settings are not configured.
  You must either define the environment variable DJANGO_SETTINGS_MODULE or call settings.configure() before accessing settings.

python manage.py 로 하기:

$ python manage.py dumpdata authentication.authority

flush inspectdb

loaddata

django-admin loaddata fixture [fixture ...]

$ python manage.py loaddata initial_data.json

loaddata 시 dumpdata 된 utf8 문자열이 아닌 경우 error 가 발생할 수 있다:

import json

filename = 'initial_data.json'

f = open(f'./{filename}', 'r')
text = f.readlines()[0]
f.close()

new_f = open(f'./{filename}', 'a')
new_f.write(str(json.loads(text)))
new_f.close()
