User Authentication in Django

Table of contents

  • Overview
  • Installation
  • Usage
  • References

Django는 사용자 인증 시스템과 함께 제공됩니다. 사용자 계정, 그룹, 권한 및 쿠키 기반 사용자 세션을 처리합니다. 문서의 이 섹션에서는 프로젝트의 필요에 맞게 기본 구현을 확장 및 사용자 정의하는 방법뿐만 아니라 기본 구현이 어떻게 작동하는지 설명합니다.

Overview

Django 인증 시스템은 authentication(인증) 과 authorization(권한 부여)을 모두 처리합니다. 간단히 말해서 authentication은 사용자가 자신이 주장하는 사람임을 확인하고 authorization은 인증된 사용자가 수행할 수 있는 작업을 결정합니다. 여기서 authentication 용어는 두 작업을 모두 가리키는 데 사용됩니다.

인증 시스템은 다음과 같이 구성되어 있습니다: - Users - Permission(사용 권한): 사용자가 특정 작업을 수행 할 수 있는지 여부를 지정하는 이진 (예/아니요) 플래그. - Groups: 둘 이상의 사용자에게 라벨과 권한을 적용하는 일반적인 방법입니다. - 구성 가능한 암호 해시 시스템 사용자 로그인 또는 콘텐츠 제한을위한 양식 및보기 도구 - 플러그 가능한 백엔드 시스템

Django의 인증 시스템은 매우 일반적인 것을 목표로 하며 웹 인증 시스템에서 흔히 볼 수있는 기능을 제공하지 않습니다. 다음과 같은 일반적인 문제에 대한 해결책이 타사 패키지에서 구현되었습니다:

  • 암호 강도 검사
  • 로그인 시도 제한
  • 제 3 자에 대한 인증 (예: OAuth)

Installation

인증 지원은 django.contrib.auth에서 Django contrib 모듈로 번들로 제공됩니다.

기본적으로 필요한 설정은 django-admin startproject에 의해 생성 된 settings.py에 이미 포함되어 있습니다.

이 설정은 INSTALLED_APPS 설정에 나열된 두 항목으로 구성됩니다:

  1. django.contrib.auth는 인증 프레임 워크의 핵심과 기본 모델을 포함합니다.
  2. django.contrib.contenttypes는 장고 내용 유형 시스템으로, 만든 모델과 관련된 권한을 허용합니다.

그리고 MIDDLEWARE 설정안의 항목:

  1. SessionMiddleware는 여러 요청에서 세션을 관리합니다.
  2. AuthenticationMiddleware는 세션을 사용하여 사용자를 요청과 연관시킵니다.

이 설정을 적용한 상태에서 manage.py migrate 명령을 실행하면 설치된 관련 앱에 정의 된 모델에 대한 인증 관련 모델 및 권한에 필요한 데이터베이스 테이블이 생성됩니다.

실습하기:

INSTALLED_APPS = [
    '...',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    '...'
]

MIDDLEWARE = [
    '...',
    'django.contrib.sessions.middleware.SessionMiddleware',
    'django.contrib.auth.middleware.AuthenticationMiddleware',
    '...'
]
