Examples

dictConfig 형식의 전체 문서는 구성 사전 로깅에 대한 최상의 정보 소스입니다.

첫째, Django logger의 모든 logging을 local file에 기록하는 간단한 구성.

  • 'filename' 경로를 Django 응용 프로그램 실행 권한있는 사용자 위치로 변경.
LOGGING = {
    'version': 1,
    'disable_existing_loggers': False,
    'handlers': {
        'file': {
            'level': 'DEBUG',
            'class': 'logging.FileHandler',
            'filename': '/path/to/django/debug.log',  # XXX: 변경
        },
    },
    'loggers': {
        'django': {
            'handlers': ['file'],
            'level': 'DEBUG',
            'propagate': True,
        },
    },
}

둘째, 로깅 시스템이 장고의 콘솔 로깅을 출력하는 방법.

  • 지역 개발 중에 유용.

  • 기본적으로이 설정은 INFO 이상의 레벨의 메세지만을 콘솔에 전송합니다 (Django의 기본 로그 설정과 동일합니다.

단, 기본값은 DEBUG = True 일 때 로그 레코드 만 표시합니다).

  • 장고는 그러한 메시지를 많이 기록하지 않습니다.

그러나 이 설정을 사용하면 DJANGO_LOG_LEVEL = DEBUG 환경 변수를 설정하여 장고의 모든 디버그 로깅을 볼 수 있습니다.

  • 모든 데이터베이스 쿼리를 포함하므로 매우 장황함.
import os

LOGGING = {
    'version': 1,
    'disable_existing_loggers': False,
    'handlers': {
        'console': {
            'class': 'logging.StreamHandler',
        },
    },
    'loggers': {
        'django': {
            'handlers': ['console'],
            # XXX: 아래 부분 에러시
            # 'level': 'DEBUG' 으로 변경해도 됨.
            'level': os.getenv('DJANGO_LOG_LEVEL', 'INFO'),
        },
    },
}
