// ...

    }

Overriding existing mapping

ux 추가 했는데 충돌나서 overriding 됨.

[W] Overriding existing mapping: 'widget.pickerfield' From 'Ext.form.field.Picker' to 'ktmss.ux.form.field.Picker'. Is this intentional?
[W] Overriding existing mapping: 'widget.pickerfield' From 'ktmss.ux.form.field.Picker' to 'Ext.form.field.Picker'. Is this intentional?

viewModel.stores 에서 TreeStore의 key 명을 treepanel로 하면 안나옴.

controller viewModel 선언안 했을 때 component.reference 와 bind.store 명 충돌.

//...

{
    xtype: 'treepanel',
    reference: 'treepanel',  // 충돌
    flex: 1,
    bind: {
        store: '{treepanel}'  // 충돌, 둘중 treePanel로 변경하면 해결.
    },
}

Ext.form.field.Picker

사용시 Ext.define 해서 사용을 권장.

하위 picker 생성시

  • createPicker() 사용하기.

  • Component는 동적 생성 권장.

  • Component의 config중 floating: true를 하면 쉽게 만들 수 있음.

Uncaught TypeError: Cannot read property '$config' of undefined

조건:

  • 하위 picker 생성.

  • 최상위 layout의 xtype이 'form', Ext.form.Panel. (단, Ext.Container 는 문제 없었음)

  • 일반적인 속성을 사용해서 만들기.

해결: Ext.form.field.Picker.createPicker() 사용.

Uncaught TypeError: Cannot read property 'maxHeight' of undefined

.createPicker() 에서 발생.

createPicker: function() {
    return {}
}

해결: 동적 로딩 사용.

createPicker: function() {
    return Ext.create('name', {});
    // or
    // return new name({});
}

viewmodel, controller 상속

상위 Component의 viewmodel, controller를 하위 Component에 강제 상속시킬수 있다.

Ext.define('AAA', {

    viewModel: 'aaa'

    controller: 'aaa',

    items: [
        {
            xtype: 'BBB',

            viewModel: 'aaa',
            controller: 'aaa',
        }
    ]
});

Ext.tab.Panel 의 Ext.tab.Tab 제어

해결: tabConfig 사용.

{
    xtype: 'replication',
    reference: 'replication',
    iconCls: 'x-fa fa-refresh',
    title: 'DB이중화',
    tabConfig: {
        /**
         * Ext.tab.Panel.onAdd.tabConfig
         * onAdd 함수에서 사용되며 oneline-doc 에는 없고 src 에만 존재함
         * 하위 Ext.tab.Tab 에 속성 부여함
         */
        reference: 'tab_replication',
        listeners: {
            scope: 'controller',
            click: 'tabclick'
        }
    }
}

Ext.window.Window 에서 lookupReference() 하려면

controller 를 반드시 지정해줘야 할 수 있다.

이유는 lookupReference 하려면 view 속성이 존재해야 함

여기서 Ext.window.Window 는 view 가 따로 없음

그런데 controller 지정함으로써 동적으로 자신이 view 라는 것이 명시되어 가능해진다.

Tree 에서 특정 데이터만 선택 가능하게 하기

Ext.tree.Panel.selectableModel 속성을 활용 할 것

javax/xml/bind/DatatypeConverter

$ sencha app build classic development

Sencha Cmd v6.5.3.6
[INF] Processing Build Descriptor : classic (development environment)
[ERR] javax/xml/bind/DatatypeConverter

Reason:

Ubuntu 18.04 LTS java ver 11 설정 되면서 sencha cmd 가 지원하지 않아 발생.

Solution:

$ su -
$ add-apt-repository ppa:webupd8team/java
$ apt-get update
$ apt-get install oracle-java8-installer
$ update-alternatives --config java

대체 항목 java에 대해 (/usr/bin/java 제공) 3개 선택이 있습니다.

  선택       경로                                       우선순상태
------------------------------------------------------------

  1        /usr/lib/jvm/java-11-openjdk-amd64/bin/java   1101      수동 모드
* 2        /usr/lib/jvm/java-8-oracle/jre/bin/java       1081      수동 모드
  3        /usr/lib/jvm/jdk1.8.0_181/bin/java            1         수동 모드
