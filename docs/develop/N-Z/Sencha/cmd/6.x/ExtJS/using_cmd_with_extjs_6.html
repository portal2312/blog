sencha generate app 명령으로 시작하여 실행중인 application으로

끝나는 Ext JS 6 application과 함께 Sencha Cmd를 사용하는 과정을 안내.

Sencha Cmd의 기대에 부합하도록 기존의 (Sencha Cmd가 아닌) application을

upgrade 하는 process는이 설명서의 끝 부분에 설명되어 있음.

먼저 "이상적인" 또는 "default" 구조를 비교의 포인트로 이해하는 것이 중요.

이 기본 구조와 기존 application의 구조 사이의 차이점은

기존 응용 프로그램의 upgrade process를 주도하는 것임.

Prerequistes

권고:

  • Introduction to Sencha Cmd

Generating Your Application (생성)

Commercial Edition을 구입했다면 ExtJS ZIP 아카이브를 download하고 첫 번째 명령을 사용:

sencha -sdk /path/to/ext6 generate app MyApp /path/to/my-app

Trial 평가판 사용시(동시 download 됨):

$ sencha generate app -ext MyApp /path/to/my-app

위의 명령 중 하나에 의해 생성 된 application 파일은 다음과 같은 구조를 갖음.

.sencha/                        # Sencha-specific files (primarily configuration)
    app/                        # Application-specific content
        sencha.cfg              # Application configuration file for Sencha Cmd
        Boot.js                 # Private, low-level dynamic loader for JS and CSS
        Microloader.js          # Loads app based on app.json content
        build-impl.xml          # Standard application build script
        *-impl.xml              # Implementations of various build phases
        defaults.properties     # Default values and docs for build properties
        ext.properties          # Build property values specific to Ext JS
        *.defaults.properties   # Build properties by environment (e.g. "testing")
        plugin.xml              # Application-level plugin for Sencha Cmd
        codegen.json            # Data for merging generated code during upgrade
    workspace/                  # Workspace-specific content (see below)
        sencha.cfg              # Workspace configuration file for Sencha Cmd
        plugin.xml              # Workspace-level plugin for Sencha Cmd

ext/                            # A copy of the Ext JS SDK
    cmd/                        # Framework-specific content for Sencha Cmd
        sencha.cfg              # Framework configuration file for Sencha Cmd
    classic/                    # Packages related to the Classic Toolkit
        classic/                # Ext JS Classic Toolkit package
        theme-neptune/          # Classic Toolkit Theme Package for Neptune
        theme-triton/           # Classic Toolkit Theme Package for Triton
        ...
    modern/                     # Packages related to the Modern Toolkit
        modern/                 # Ext JS Modern Toolkit package
        theme-neptune/          # Modern Toolkit Theme Package for Neptune
        theme-triton/           # Modern Toolkit Theme Package for Triton
        ...
    packages/                   # Framework supplied packages
        charts/                 # Charts package
        ux/                     # Contents of "Ext.ux" namespace
    ...

index.html                      # The entry point to your application
app.json                        # Application manifest
app.js                          # Launches the Application class
app/                            # Your application's source code in MVC structure
    model/                      # Folder for application model classes
    store/                      # Folder for application stores
    view/                       # Folder for application view classes
        main/                   # Folder for the classes implementing the Main View
            Main.js             # The Main View
            MainModel.js        # The `Ext.app.ViewModel` for the Main View
            MainController.js   # The `Ext.app.ViewController` for the Main View
    Application.js              # The `Ext.app.Application` class

packages/                       # Sencha Cmd packages
workspace.json                  # Workspace JSON descriptor

build/                          # The folder where build output is placed

Building Your Application

마크업 페이지와 javascript 코드, Sass 및 테마 파일을 "build" 폴더에 생성.

$ sencha app build

application build 를 위해 application root 폴더에서 실행.

중요: 반드시 application 폴더의 top-level 폴더 위치에서 이 명령어를 실행해야 함. (path/to/MyApp)

중요: sencha app 명령어 사용시 -sdk 파라미터 지정하지 말 것.

이 명령어는 application root 폴더에서 실행되어야 하기 때문에

sencha 명렁어는 어떤 SDK를 사용해야 하는지 알고 있으며,

-sdk 파라미터 사용하면 sencha 명령어는 현재 Dir을 application root가 아닌

SDK 지정 폴더로 인식함.

Development Mode

bootstrap

  • app.json + application source 파일을 참고해서 bootstrap 생성.

  • 수집된 정보를 dynamic class loader(Ext.Loader 및 Microloader 지칭)로

전달해주기 때문에 별도로 관리할 필요가 없게 함.

그러나 javascript code file name 변경, 이동 또는 styling의 변경은 bootstrap 정보

또는 생성된 CSS 정보의 일치성을 무효화시켜 application 실행에 문제가 됨.

bootstrap update 방법 두가지: 아래 내용 부터

Sencha app watch

$ sencha app watch

  • development build + command prompt로 돌아와 build한 file 변화를 모니터링.

  • Fashion 활성화: 실시간 CSS 반영, Live Update

  • 확인하기

  $ http://localhost:1814
  • watch 종료하기: ctrl + c

Development Build and Refresh

bootstrap, CSS 수동 update시:

$ sencha app build development
$ sencha app refresh

app build development: refresh + updated CSS file 생성을 위해 styling도 compile 하게 됨.

refresh: javascript update.

Extending Your Application

sencha generate: Control 및 Model을 사용한 대표적인 MVC component를 생성.

$ sencha help generate

# Sencha Cmd v6.2.1.29
# sencha generate

# This category contains code generators used to generate applications as well
# as add new classes to the application.

# Commands
#   * app - Generates a starter application
#   * controller - Generates a Controller for the current application
#   * form - Generates a Form for the current application (Sencha Touch Specific)
#   * model - Generates a Model for the current application
#   * package - Generates a starter package
#   * profile - Generates a Profile for the current application (Sencha Touch Specific)
#   * theme - Generates a theme page for slice operations (Ext JS Specific)
#   * view - Generates a View for the current application (Ext JS Specific)
#   * workspace - Initializes a multi-app workspace

중요: 앞으로 설명할 명령어는 반드시 application root Dir(= /path/to/MyApp) 실행.

Generating Models

# pwd /home/mky/myapp
$ sencha generate model User id:int,name,email

app/
    model/
          User.js

Generating Views

# pwd /home/mky/myapp
$ sencha generate view foo.Thing

app/
    view/
        foo/                    # 새로운 뷰 클래스 구현을 위한 폴더
            Thing.js            # 새로운 뷰
            ThingModel.js       # 뷰 모델 `Ext.app.ViewModel`
            ThingController.js  # 뷰 컨트롤러 `Ext.app.ViewController`

뷰 이름 뒤에 아무런 파라미터가 사용되지 않았음에 주의. 그러나, 만약 필요하다면 base 클래스를 추가할 수도 있음.

# pwd /path/to/MyApp
$ sencha generate view -base Ext.tab.Panel foo.Thing

Ext.tab.Panel class 기반으로 extend 된 view class 생성됨.

Generating Controllers

Ext JS 5+에서 Sencha Cmd에 의해 생성된 각각의 view는 Ext.app.ViewController를 기본으로 갖기 때문에 대부분의 경우 Ext.app.Controller를 기반으로 한 global controller를 생성할 필요가 없다.

만약 새로운 controller가 필요하다면 model 및 view와 유사한 방식으로 생성할 수 있음.

# pwd /path/to/MyApp
$ sencha generate controller Central  # XXX: Central == <controller-name>

app/
    controller/
               Central.js

controller name 뒤에 paramater 없음.

Customizing The Build

build 사용자화(Customize) 하기 위해 제일 처음 볼 곳은 app.json 임.

파일은 많은 옵션들과 이와 관련된 주석들을 포함하고 있음.

추가 설정은 'workspace.json' 파일임.

Going Further (추가예정)

.json 파일 이외의 설정을 건드리는 것은 권한하지 않음.

일부의 경우에는 필요할 수도 있음.

만약 application 또는 workspace 폴더 내에 있는 .sencha 폴더가 포함하고 있는

파일들을 수정하면, 설정이 지원되지 않을 수 있음.

(수정허용안함: .sencha/app/sencha.cfg, .sencha/workspace/sencha.cfg, .sencha/app/*.properties)

Sencha Support는 앞서 설명한 app.json이나 workspace.json 파일만을 수정해서 문제를

해결할 수 있다고 보고 관련된 사례를 수집하고 있음.

The classpath

sencha app build: app.json 에 저장된 classpath 속성을 통해 application 소스가

저장된 위치를 식별함. 기본값:

# app.json

"classpath": [
    "app",
    "${toolkit.name}/src"
]

classpath dir 추가: compiler에게 application build를 위해 필요한 소스코드 위치 추가.

Upgrading Your Application

generate app 생성된 기본 application은 Sencha Cmd에 연계된 두가지 기본 컨텐츠,

즉 build script(또는 scaffolding)과 Sencha SDK에 의해 사용되는 중요 컨텐츠를 포함.

따라서, 이 두 조각을 필요에 따라 upgrade할 수 있으며, 다음 명령어를 사용.

$ sencha app upgrade [path-to-new-framework]

"path-to-new-framework" 선택적 적용.

Sencha Cmd와 application에 의해 사용되는 framework를 둘다 upgrade시.

참조: Understand App Upgrade

Porting Application to Sencha Cmd

Sencha Cmd 의해 생성된 스케폴드 핵심 파일들.

.sencha/
app.json
build.xml
index.html

이 파일들 중 처음 3개는 application으로부터 생성된 temp 폴더로 단순 복사가능.

기존 application 일반적으로 진입 페이지를 갖고 있을텐데 만약 'index.html'이

아니라면 app.json 파일에 추가하면 됨.

{
    ...
    "indexHtmlPath": "index.php"
}

당연하겠지만, 이 값은 application 마다 다를 수 있음.

정확한 값으로 대처해야 함.

생성된 build script가 이 markup 파일을 이해 할 수 있도록 하려면 진입파일에는

Sencha Cmd가 생성해준 index.html 파일에 포함된 다음의 표준 문항을 포함해야 함.

<script id="microloader" type="txt/javascript" src="bootstrap.js"></script>

development mode에서 이 microloader는

sencha app build 또는 sencha app refresh 가 수행되는 동안 Sencha Cmd가 생성한

파일을 loading.

sencha app build 는 refresh를 수행할 때 한번에 포함되어 수행되므로

sencha app refresh 후 sencah app build 를 따로 수행할 필요는 없다.

Configuration

아마 기존 application과 기본 구조로 생성된 Sencha Cmd application은 몇가지가

일치하지 않을 것이다. 이 문제 해결위해 두가지.

  • Sencha Cmd 가 생성한 구조와 일치하도록 기존 application 구조를 조정

  • 기존 application 구조에 일치하도록 build process를 설정

파일들을 수정하면, 설정이 지원되지 않을 수 있음.

(수정허용안함: .sencha/app/sencha.cfg, .sencha/workspace/sencha.cfg, .sencha/app/*.properties)

Sencha Support는 앞서 설명한 app.json이나 workspace.json 파일만을 수정해서 문제를

해결할 수 있다고 보고 관련된 사례를 수집하고 있음.

The classpath

sencha app build: app.json 에 저장된 classpath 속성을 통해 application 소스가

저장된 위치를 식별함. 기본값:

# app.json

"classpath": [
    "app",
    "${toolkit.name}/src"
]

classpath dir 추가: compiler에게 application build를 위해 필요한 소스코드 위치 추가.

Upgrading Your Application

generate app 생성된 기본 application은 Sencha Cmd에 연계된 두가지 기본 컨텐츠,

즉 build script(또는 scaffolding)과 Sencha SDK에 의해 사용되는 중요 컨텐츠를 포함.

따라서, 이 두 조각을 필요에 따라 upgrade할 수 있으며, 다음 명령어를 사용.

$ sencha app upgrade [path-to-new-framework]

"path-to-new-framework" 선택적 적용.

Sencha Cmd와 application에 의해 사용되는 framework를 둘다 upgrade시.

Porting Application to Sencha Cmd

Sencha Cmd 의해 생성된 스케폴드 핵심 파일들.

.sencha/
app.json
build.xml
index.html

이 파일들 중 처음 3개는 application으로부터 생성된 temp 폴더로 단순 복사가능.

기존 application 일반적으로 진입 페이지를 갖고 있을텐데 만약 'index.html'이

아니라면 app.json 파일에 추가하면 됨.

{
    ...
    "indexHtmlPath": "index.php"
}

당연하겠지만, 이 값은 application 마다 다를 수 있음.

정확한 값으로 대처해야 함.

생성된 build script가 이 markup 파일을 이해 할 수 있도록 하려면 진입파일에는

Sencha Cmd가 생성해준 index.html 파일에 포함된 다음의 표준 문항을 포함해야 함.

<script id="microloader" type="txt/javascript" src="bootstrap.js"></script>

development mode에서 이 microloader는

sencha app build 또는 sencha app refresh 가 수행되는 동안 Sencha Cmd가 생성한

파일을 loading.

sencha app build 는 refresh를 수행할 때 한번에 포함되어 수행되므로

sencha app refresh 후 sencah app build 를 따로 수행할 필요는 없다.
