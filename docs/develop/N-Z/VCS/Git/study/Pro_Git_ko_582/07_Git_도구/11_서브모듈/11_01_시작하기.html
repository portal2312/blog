서브모듈 시작하기

Table of contents

  • Summary

먼저 관리중인 프로젝트(myproject) 로 이동하여 git submodule add 저장소 URL(절대, 상태 가능)경로로 가져오기:

$ cd myproject
$ git submodule add https://github.com/apps/myapp

$ git status

On branch master
Your branch is up-to-date with 'origin/master'.

Changes to be committed:
  (use "git reset HEAD <file>..." to unstage)

    new file:   .gitmodules
    new file:   myapp

기본적으로 서브모듈은 프로젝트 저장소의 이름으로 디렉토리를 만든다.

명령의 마지막에 원하는 이름을 넣어 다른 디렉토리 이름으로 서브모듈을 추가할 수도 있다.

.gitmodules 파일은 서브디렉토리와 하위 프로젝트 URL의 매핑 정보를 담은 설정 파일 내용:

[submodule "myapp"]
    path = myapp
    url = https://github.com/apps/myapp

Git은 myapp 디렉토리를 서브모듈로 취급하기 때문에 해당 디렉토리 아래의 파일 수정사항을 직접 추적하지 않는다.

대신 서브모듈 디렉토리(myapp)를 통째로 특별한 commit 으로 취급한다.

git diff --cached myapp:

$ git diff --cached myapp

diff --git a/myapp b/myapp
new file mode 160000
index 0000000..ae07c16
--- /dev/null
+++ b/myapp
@@ -0,0 +1 @@
+Subproject commit ae07c166978bb396539b9eb437fd8c13220c5336

git diff --cached --submodule 명령어는 서브모듈 자세히 보기:

$ git diff --cached --submodule

diff --git a/.gitmodules b/.gitmodules
index dedf31f..b08b932 100644
--- a/.gitmodules
+++ b/.gitmodules
@@ -1,3 +1,6 @@
 [submodule "menu_mgmt"]
        path = menu_mgmt
        url = http://10.40.223.88/apps/menu_mgmt
+[submodule "myapp"]
+       path = myapp
+       url = http://0.0.0.0/mkkim/myapp
Submodule myapp 0000000...ae07c16 (new submodule)

이제 하위 프로젝트를 포함한 커밋을 생성하면 아래와 같은 결과를 확인할 수 있다:

$ git commit -am 'added myapp module'

[master fb9093c] added myapp module
 2 files changed, 4 insertions(+)
 create mode 100644 .gitmodules
 create mode 160000 myapp

Summary

Project 에 submodule 추가하기:

$ cd myproject
$ git submodule add <submodule-url>

submodule 관리 파일 .gitmodules:

[submodule "myapp"]
    path = myapp
    url = https://github.com/apps/myapp

cached 영역 이력 보기:

$ git diff --cached <submodule-name>
$ git diff --cached --submodule  # 전체
