서브모듈 수정 사항 공유하기

Table of contents

  • Summary

Remote myapp(b785fe9):

* 2018-07-23 13:27:07 +0900 (2시간 전) b785fe9 mkkim  (HEAD -> stable, origin/stable) -- Create ./log/180722.log

Local myproject - myapp(b785fe9-6eb9948):

* 2018-07-23 14:59:11 +0900 (41초 전) 6eb9948 mkkim  (HEAD -> stable) -- Update README.md
* 2018-07-23 13:27:07 +0900 (2시간 전) b785fe9 mkkim  (origin/stable) -- Create ./log/180722.log

현재 서브모듈은 변경된 내용을 포함하고 있다.

이 중 일부는 서브모듈 자체를 업데이트하여 Upstream 저장소에서 가져온 것이고 일부는 로컬에서 직접 수정한 내용이다.

로컬에서 수정한 것은 아직 공유하지 않았으므로 아무도 사용할 수 없는 코드이다.

$ cd myproject
$ git diff

Submodule myapp b8d6bc7..6eb9948:
  > Update README.md
  > Create ./log/180722.log

서브모듈의 변경사항을 Push 하지 않은 채로 메인 프로젝트에서 커밋을 Push 하면 안 된다.

변경 사항을 Checkout 한 다른 사람은 서브모듈이 의존하는 코드를 어디서도 가져올 수 없는 상황이 돼 곤란해진다.

서브모듈의 변경사항은 우리의 로컬에만 있다.

이런 불상사가 발생하지 않도록 하려면 메인 프로젝트를 Push 하기 전에 서브모듈을 모두 Push 했는지 검사하도록 Git에게 물어보면 된다.

git push 명령에 --recurse-submodules= 옵션을 주고 이 옵션의 값으로 check나 on-demand를 설정한다.

--check 는 간단히 서브모듈의 로컬 커밋이 Push 되지 않은 상태라면 현재의 Push 명령도 실패하도록 하는 옵션이다.

그런데 Local myproject 변경 내용 없는 경우:

$ git push --recurse-submodules=check
Everything up-to-date

먼저 Local myproject 에서 수정된 내용이 있어야 한다:

$ cd myproject
$ vi README.md
$ git commit -avm 'Update README.md'

* 2018-07-23 15:03:08 +0900 (2초 전) 3b043d4 mkkim  (HEAD -> master) -- Update README.md/#4
* 2018-07-23 14:46:04 +0900 (17분 전) 28aa91e mkkim  (origin/master, origin/HEAD) -- Update README.md/#3

$ git push --recurse-submodules=check

다음 하위 모듈 경로에 리모트 어디에도 없는
변경 사항이 있습니다:
  myapp

리모트에 푸시하려면

        git push --recurse-submodules=on-demand

또는 해당 경로로 cd한 다음

        git push

명령을 시도해 보십시오

fatal: 중지함.
fatal: The remote end hung up unexpectedly

가장 단순한 해결 방법은 각 서브모듈 디렉토리로 가서 직접 일일이 Push를 해서 외부로 공유하고 나서 메인 프로젝트를 Push 하는 것이다.

이 옵션이 항상 적용되도록 하고 싶으면 git config push.recurseSubmodules check 명령으로 설정한다.

옵션으로 설정할 수 있는 다른 값으로 on-demand 값이 있는데, 이 값으로 설정하면 Git이 Push를 대신 시도한다:

$ cd myproject
$ git config push.recurseSubmodules check

위에서 보듯이 Git이 메인 프로젝트를 Push 하기 전에 DbConnector 모듈로 들어가서 Push를 한다. 모종의 이유로 서브모듈 Push에 실패한다면 메인 프로젝트의 Push 또한 실패하게 된다. git config push.recurseSubmodules on-demand 명령으로 설정할 수 있다.

$ git push --recurse-submodules=on-demand

실패시:

Pushing submodule 'myapp'
Username for 'http://0.0.0.0': mkkim
Password for 'http://mkkim@0.0.0.0':
warning: http://0.0.0.0/mkkim/myapp.git/(으)로 리다이렉트
Everything up-to-date
다음 하위 모듈 경로에 리모트 어디에도 없는
변경 사항이 있습니다:
  myapp

리모트에 푸시하려면

        git push --recurse-submodules=on-demand

또는 해당 경로로 cd한 다음

        git push

명령을 시도해 보십시오

fatal: 중지함.
fatal: The remote end hung up unexpectedly

$ cd myapp
(stable)$ git push
(stable)$ cd ..
$ git push --recurse-submodules=on-demand

위에서 보듯이 Git이 메인 프로젝트를 Push 하기 전에 myapp 모듈로 들어가서 Push를 한다.

모종의 이유로 서브모듈 Push에 실패한다면 메인 프로젝트의 Push 또한 실패하게 된다.

git config push.recurseSubmodules on-demand 명령으로 설정할 수 있다.

$ cd myproject
$ git config push.recurseSubmodules on-demand
