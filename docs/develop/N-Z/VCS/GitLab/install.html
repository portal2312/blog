Ubuntu

https://about.gitlab.com/installation/#ubuntu

1. Install and configure the necessary dependencies

sudo apt-get update
sudo apt-get install -y curl openssh-server ca-certificates

postfix: GitLab 이 default 로 사용하는 SMTP server

외부 SMTP server 설정: https://docs.gitlab.com/omnibus/settings/smtp.html

sudo apt-get install -y postfix

Postfix 설치 구성 화면 - 인터넷 사이트 선택하기. - 메일 이름에 서버의 외부 DNS를 사용 (Enter key). - 추가 화면이 나타나면 계속 (Enter key)를 눌러 기본값을 적용하기.

2. Add the GitLab package repository and install the package

GitLab 패키지 저장소를 추가하십시오:

curl https://packages.gitlab.com/install/repositories/gitlab/gitlab-ee/script.deb.sh | sudo bash

그런 다음 GitLab 패키지를 설치하십시오. http://gitlab.example.com 을 GitLab 인스턴스에 접근하고자하는 URL로 변경하십시오. 설치가 자동으로 해당 URL에서 GitLab을 구성하고 시작합니다. HTTPS 는 설치 후 additional configuration이 필요합니다.

3. Browse to the hostname and login

처음 방문하면 비밀번호 재설정 화면으로 리디렉션됩니다. 초기 관리자 계정에 대한 비밀번호를 입력하면 로그인 화면으로 다시 리디렉션됩니다. 기본 계정의 사용자 이름 root 를 사용하여 로그인하십시오.

설치 및 구성에 대한 자세한 지침은 설명서를 참조하십시오.

4. Set up your communication preferences

통신 환경 설정하기

이메일 구독 환경 설정 센터를 방문하여 귀하와 연락할 시기를 알려주십시오. 우리는 당신에게 이메일을 보내는 방법과 빈도를 완벽하게 통제 할 수 있도록 명시적인 이메일 옵트 인 정책을 가지고 있습니다.

한 달에 두 번, 우리는 새로운 기능, 통합, 문서 및 개발자 팀의 배경 이야기를 포함하여 알아야 할 GitLab 뉴스를 발송합니다. 버그 및 시스템 성능과 관련된 치명적인 보안 업데이트를 원하시면 전용 보안 뉴스 레터에 가입하십시오.

중요 정보 : 보안 뉴스 레터를 수신하지 않으면 보안 경고가 수신되지 않습니다.
