ldconfig는 runtime때 여러 위치에 존재하는 shared object, 쉽게말해 동적 라이브러리를 연결해주는 dynamic linker의 설정프로그램이라고 보시면 됩니다.

여기서 runtime dynamic linker는 /lib/ld.so 를 말하는 겁니다.

어떤 shared object가 필요한지는 실행파일이 말해줍니다.

예:

zen@mkkimpc:~$ ldd /bin/ls

  linux-vdso.so.1 =>  (0x00007fff2ffcd000)
  libselinux.so.1 => /lib/x86_64-linux-gnu/libselinux.so.1 (0x00007f0988007000)
  libc.so.6 => /lib/x86_64-linux-gnu/libc.so.6 (0x00007f0987c40000)
  libpcre.so.3 => /lib/x86_64-linux-gnu/libpcre.so.3 (0x00007f09879cd000)
  libdl.so.2 => /lib/x86_64-linux-gnu/libdl.so.2 (0x00007f09877c9000) /lib64/ld-linux-x86-64.so.2 (0x0000557e991fc000)
  libpthread.so.0 => /lib/x86_64-linux-gnu/libpthread.so.0 (0x00007f09875ab000)

ls가 동작하기 위해 필요한 so들 목록이 나오고, 해당 so파일을 어느 경로에 있는걸 가져올껀지도 보여줄껍니다.

/etc/ld.so.conf는 이 경로를 지정해주는 설정파일이라고 보시면 편합니다.

not found

해당 라이브러리 경로를 추가한다

$ su -
$ export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/usr/local/lib

Reference

man ldconfig
