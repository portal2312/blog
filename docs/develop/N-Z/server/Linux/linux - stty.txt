unix, linux, 네트워크 장비중 설정에 stty 가 있습니다.

stty 설정으로 backspase 설정을 변경 할 수 있습니다.

그 뿐만 아니라 여러가지 Key의 설정을 할 수 있습니다.

개인 계정마다 줄 수 도 있고 전역설정으로 모든 사용자들에게 적용할 수 있습니다.

대신증권에서는 발생된 내용으로는 바로(nOM을 통하지 않는) 접속을 하면 backspace 로 만 삭제가 된다 라고 하셨고

nOM 을 통한 접속은 shift + backspace 하면 된다 하셨습니다.

이렇게 되면 해당 접속한 계정의 stty erase Code 값을 알아야 됩니다.

확인할 방법은 접속한 계정으로 stty -a | grep erase 하면 됩니다.

그리고 전역 설정이나 개인 설정으로 서로 다르게 되어 문제가 발생했는지도 알아봐 주셔야 합니다.
예) root 에서 erase ^H;  그런데 hts에서는 erase ^?; 가 서로 다른 경우 (;는 구분자임으로 무시)

만약 다르다면 어느 것을 기준으로 둘지도 알려주셔야 합니다.

확인이 되면 데몬 소스 수정 해야됩니다.

keyboard event 값 입력 들어오는 소스 부분에 해당 erase Code 와 일치시 삭제하는 코드로 바꿔치기 할 겁니다.

저는 테스트는 191에서 진행했습니다. 그리고 테스트 화면은 같이 보면 좋겠습니다. 연락주십시오.
