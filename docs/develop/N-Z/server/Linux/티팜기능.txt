티팜

티시스에서 작용된 사례 
- 접근제어와 티팜을 연동하여 접근제어 화면에서
  서버의 패스워드를 요청하면 티팜을 통해 패스워드
  확득하여 접근제어 화면에서 확인하여 사용자는 패스워드를 입력하여 로그인함
(접근제어에서도 패스워드 변경및 자동로그인 가능하지만 윈도우 패스워드 변경 미지원으로 티팜을  윈도우 중심으로 활용)

                                                     
티팜 주요 기능

1.사용자는 자동로그인 으로 시스템 접속(접속시마다 시스템 계정은 변경이됨)                  //접근제어에서 자동로그인 가능하며 패스워드 변경주기를 접속기준 으로는 안되어 있음

2.패스워드는 암호화하여 안전하게 보관 (AES256 암호화 프로토콜 사용)                        //접근제어 기능 제공

3.계정 패스워드의 불출 관리 (발급 이력 저장)                                               //접근제어 기능 제공

4.사용자에게 패스워드 발급을 위한 승인단계는 0회 이상으로 설정 가능 (0회=자동승인)         //접근제어 기능 제공
?사용자는 승인된 기간 동안 제공받은 패스워드로 대상 서버에 여러 번 접속이 가능            //접근제어 기능 제공(시스템 사용기간 설정으로 기능 제공)
?사용자의 사용 승인 기간이 종료되면 TPAM은 해당 계정의 패스워드를 자동으로 변경           //접근제어에서 시스템 계정 변경은 사용자의 승인기간과 연동되어 있지 않으며 따로 패스워드변경주기 등록함 
?사용자의 접속 여부와 상관없이 TPAM은 관리하는 모든 계정의 패스워드를 주기적으로 변경     //접근제어 기능 제공

5 요청자의 패스워드 요청, 승인자의 승인/거부의 워크플로우 지원                             //접근제에에서 사용자가 패스워드 요청 기능은 없음(관리자만 확인가능)

6.요청, 승인, 사용 이력에 대한 보고서 지원                                                 //보고서 개발 필요

7.유닉스,리눅스,윈도우계열,db계정 패스워드 관리 가능                                       //접근제어에서 윈도우,db계정 패스워드 변경 기능 미지원
