"gzip"메소드를 사용하여 응답을 압축하는 필터입니다.

이는 종종 전송 된 데이터의 크기를 절반 이상 줄이는데 도움이 됩니다.

Example Configuration

gzip            on;
gzip_min_length 1000;
gzip_proxied    expired no-cache no-store private auth;
gzip_types      text/plain application/xml;

$gzip_ratio 변수는 달성 된 압축 비율을 기록하는 데 사용할 수 있습니다.

Directives (지시문)

gzip

Syntax: gzip on | off;
Default: gzip off;
Context: http, server, location, if in location

응답의 gzipping을 활성화 또는 비활성화합니다.

gzip_buffers

Syntax: gzip_buffers number size;
Default: gzip_buffers 32 4k|16 8k;
Context: http, server, location

응답을 압축하는 데 사용되는 버퍼의 수 와 크기 를 설정합니다.

기본적으로 버퍼 크기는 하나의 메모리 페이지와 동일합니다.

이것은 플랫폼에 따라 4K 또는 8K입니다.

  • 버전 0.7.28 이전까지 4 개의 4K 또는 8K 버퍼가 기본적으로 사용되었습니다.

gzip_comp_level

Syntax: gzip_comp_level level;
Default: gzip_comp_level 1;
Context: http, server, location

응답의 gzip 압축 수준을 설정합니다. 사용할 수있는 값의 범위는 1에서 9 사이입니다.

gzip_disable

Syntax: gzip_disable regex ...;
Default: —
Context: http, server, location
This directive appeared in version 0.6.23.

지정된 정규 표현식 중 하나와 일치하는 "User-Agent"헤더 필드가있는 요청에 대한 응답의 gzipping을 비활성화합니다.

특수 마스크 "msie6"(0.7.12)은 정규식 "MSIE [4-6] ."에 해당하지만 더 빠르게 작동합니다.

버전 0.8.11부터 "MSIE 6.0; ... SV1 "은이 마스크에서 제외됩니다.

gzip_min_length

Syntax: gzip_min_length length;
Default: gzip_min_length 20;
Context: http, server, location

gzip으로 처리할 응답의 최소 길이를 설정합니다.

길이는 "Content-Length"응답 헤더 필드에서만 결정됩니다.

gzip_http_version

Syntax: gzip_http_version 1.0 | 1.1;
Default: gzip_http_version 1.1;
Context: http, server, location

응답 압축에 필요한 요청의 최소 HTTP 버전을 설정합니다.

gzip_proxied

Syntax: gzip_proxied off | expired | no-cache | no-store | private | no_last_modified | no_etag | auth | any ...;
Default: gzip_proxied off;
Context: http, server, location

요청 및 응답에 따라 프록시 요청에 대한 응답의 gzipping을 활성화하거나 비활성화합니다. 요청이 프록시된다는 사실은 "Via"요청 헤더 필드의 존재에 의해 결정됩니다. 이 지시문은 여러 매개 변수를 허용합니다.

gzip_types

Syntax: gzip_types mime-type ...;
Default: gzip_types text/html;
Context:http, server, location

"text/html"외에도 지정된 MIME 유형에 대한 응답의 gzipping을 가능하게합니다.

특수 값 "*"은 모든 MIME 유형 (0.8.29)과 일치합니다.

"text/html"유형의 응답은 항상 압축됩니다.

gzip_vary

Syntax: gzip_vary on | off;
Default: gzip_vary off;
Context: http, server, location

directive gzip, gzip_static 또는 gunzip이 활성화 된 경우 "Vary : Accept-Encoding"응답 헤더 필드 삽입을 활성화하거나 비활성화합니다.

Embedded Variables

$gzip_ratio

달성 된 압축 비율은 원본 및 압축 된 응답 크기 간의 비율로 계산됩니다.
