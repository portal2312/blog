서브모듈 포함한 프로젝트 Clone

Table of contents

  • Summary

myproject 를 clone 하여 가져오면 myapps 은 빈 디렉토리이다:

$ git clone https://github.com/projects/myproject
$ cd myproject
$ cd myapp
$ ls
$

git submodule init 명령을 실행하면 서브모듈 정보를 기반으로 로컬 환경설정 파일이 준비된다.

git submodule update 명령으로 이후 서브모듈의 리모트 저장소에서 데이터를 가져오고 서브모듈을 포함한 프로젝트의 현재 스냅샷에서 Checkout 해야 할 커밋 정보를 가져와서 서브모듈 프로젝트에 대한 Checkout을 한다.

$ cd myproject
$ git submodule init

Submodule 'myapps' (https://github.com/apps/myapp) registered for path 'myapps'

$ git submodule update

Cloning into 'myapp'...
remote: Counting objects: 11, done.
remote: Compressing objects: 100% (10/10), done.
remote: Total 11 (delta 0), reused 11 (delta 0)
Unpacking objects: 100% (11/11), done.
Checking connectivity... done.
Submodule path 'myapp': checked out 'c3f01dc8862123d317dd46284b05b6892c7b29bc'

$ cd menu_mgmt/

((51eb7a3...))$ ls

git clone --recurse <project-url> 명령으로 clone 한번에(서브모듈 자동 초기화 + 업데이트):

$ git clone --recurse https://github.com/projects/myproject.git
$ cd myproject
$ ls

README.md  menu_mgmt

$ cd menu_mgmt/

((51eb7a3...))$ ls

Summary

project clone 후 submodule 동기화하기:

$ git clone <project-url> <project-name>
$ cd <project-name>

$ git submodule init; git submodule update

or

$ git submodule update --init

project clone + submodule 동기화하기:

$ git clone --recurse-submodules <project-url> <project-name>
