버전관리

수정, 복구, 이력관리, 비교.

(Local) Version Control System (VCS - 로컬 버전 관리)

Database를 통한 단순 저장.

Central Version Control System (CVCS - 중앙 집중식 버전 관리)

  • Client들이 Central Server로 저장시킴.

  • 단점: 중앙 서버에서 문제 발생시.

Distributed Version Control System (DVCS - 분산 버전 관리)

Client, Server 모두 저장 및 공유.

짧게 보는 Git의 역사 (장점)

  • 빠른 속도
  • 단순한 구조
  • 비선령적인 개발
  • 완벽한 분산
  • Linux Kernel 같은 대형 Projects 에도 유용할 것 (Data Size면에서)
  • Branch (Git 기능중 강점)

Git 기초

차이가 아니라 스냅샷

[Git] 모든 상황을 저장 (Snapshot) <> [VCS] File별 변화에 따른 시간순으로 관리.

거의 모든 명령을 로컬에서 실행

Git의 무결성

  • 데이터 저장전 항상 Checksum 을 구하고 해당 Checksum 으로 데이터 관리.

  • Checksum 은 Git의 Atomic 데이터, 없으면 관리 안됨.

  • Git은 SHA-1 hash를 이용하여 Checksum 을 생성. (File 내용, Dir 구조를 이용)

  • Git은 File 명을 저장하지 않고 hash 를 저장.

Git은 데이터를 추가할 뿐

3가지 상태 (중요)

  • Working Directory

  • Staging Area

  • .git dir (Repository)

[Working Dir]    [Staging Area]        [.git Dir] (Repository)

file1   -> fixed -> * Modified                       -> * Committed
                 -> git add <file1>
                 -> * Staged           -> git commit

        <-------------- Check out -----------------

CLI

CLI 는 GUI 를 다룰수 있음.

GUI 는 CLI 를 할 수 없음. (명령어 제약 등등)

Git install

생략

Git 최초 설정

  • 설정한 내용은 Upgrade 해도 유지, 언제든 변경 가능.

  • 적용순서는 작은 단위가 먼저.

  • Linux

File Description
/etc/gitconfig ALL (User, Dir)
~/.gitconfig User
~/.config/git/config User
.git/config Dir

  • Windows

    • $HOME Dir에서 찾음.

사용자 정보

  • 설치 후 가장 먼저 해야 됨.

  • 이후 commit 후에 정보를 변경할 수 없음.

git config --global user.name 'mkkim'
git config --global user.email 'portal2312@gmail.com'
  • 프로젝트별 다른 Name, Email 을 사용하려면 --global 옵션해제.

편집기

git config --global core.editor vim

설정 확인

git config --list
...

user.name=mkkim
user.email=portal2312@gmail.com


git config user.name
...

user.name=mkkim

도움말

git help

git <Tab-key>
