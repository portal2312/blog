Versions

vi /opt/gitlab/version-manifest.txt

Installation

https://about.gitlab.com/downloads/#centos7

  1. dependencies

    $ sudo yum install curl policycoreutils openssh-server openssh-clients
$ sudo systemctl enable sshd
$ sudo systemctl start sshd
$ sudo yum install postfix
$ sudo systemctl enable postfix
$ sudo systemctl start postfix
$ sudo firewall-cmd --permanent --add-service=http
$ sudo systemctl reload firewalld

  2. Add the GitLab package server and install the package

  • GitLab 패키지 서버를 추가 하고 yum 으로 설치.

    curl -sS https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.rpm.sh | sudo bash
sudo yum install gitlab-ce

  • 위 방법을 원하지 안는 경우 아래와 같이 rpm Downlaod 해서 설치. bash curl -LJO https://packages.gitlab.com/gitlab/gitlab-ce/packages/el/7/gitlab-ce-XXX.rpm/download rpm -i gitlab-ce-XXX.rpm

  1. Configure and start GitLab bash gitlab-ctl reconfigure

  2. Browse to the hostname and login ```bash go to https://localhost web page open.

    change your passwd: ****

    id/pw: root/**** ```

  3. Profile configure

  • 계정에 대한 Name, Email 등 수정후 save 시 해당 Email 인증하기. (스팸함 확인)
  1. Projects Add
  • UI 에서 만들면 해당 Projects 에 맞게 명령어를 보여줌, 해당 상황에 맞게 실행하기.
  1. Profile

  • Name: 실제 로그인시 사용되는 이름, 중요 (= userid)

  • Username: 형식적인 이름.

  • Email: 알림 및 로그인 등 사용, 중요.

  1. 사용자 권한

  • 사용자는 원하는 그룹을 선택하고 Access Request 버튼을 선택한다.

  • 승인은 관리자의 email로 알림이 오고 url 선택시 승인해줄 수 있음.

    또는 요청한 해당 그룹 > Members

  1. Project - Branch protected/unprotected

  • Projects의 branch-master 에 protected 설정시 push해도 변경 안됨.

  • 관리자계정 > Projects > Protected branches > master > unprotectd > 선택 or 해제

(처음 만드는 거라면 계정 대한 권한을 Master 로 변경하면 됨)

Configuration

$ vi /etc/gitlab/gitlab.rb
  • Port ```vim ...

external_url 'http://localhost'

... ```
