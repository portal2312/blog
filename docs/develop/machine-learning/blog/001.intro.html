What is machine learning?

  • E(experience): data, 올바른 data를 갖고 tasks 를 설명 (축구: 패스, 슛, 움직임)
  • T(tasks): 명확한 목표 (축구: 게임에서 이기기)
  • P(performance measure): 효율적인 척도 (축구: 골)

경험을 통해 성능을 향상시킨다.

Task

classification: discrete target values
손으로 쓴 숫자를 머신이 인식하기 까지

regression: real target values
연속된 값을 인식하는 것, 회귀 (supervised: classification -> regression)

clustering: no target values
비연속성의 데이터를 추출하기 (unsupervised)

Performance

classification: 0-1 loss function
올바르게 인식된 경우 1

regression: L2 loss function
근사치(거리)에 이르는 값인 경우 높은 점수를 주기, 거리를 최소하 하는 것이 목적

clustering: 최적의 중심을 잡고 최적의 거리가 나오도록 하기

Experience

data 기준, data 를 기준으로 트레이닝 시키기

classification: labeled data, x1 = 5, x2 = 6

regression: labeled data, (x1,y1) = (1,2) 모델 값을 생성해서 예측

clustering: unlabeled data 데이터만 던지고 알아서 판단

Example:

기준 값을 정하여 거리의 합이 짧아지는 기울기를 계속 변경하면서 구한다.
단, 기울기만으로는 local minuma problem 발생.

예) (키, 몸무게)

overfitting: 예측하려는 모델을 만들기 위한 과정에서의 문제

L2 Regularization (중요)

데이터를 잘 설명하면서 (평균 기울기로 부터)거리를 적게 하기

  1. 현재를 잘 설명
  2. 미래를 잘 설명(너무 복잡한 모델을 피하기)

예) 100 가지 중 80 을 training 하여 모델 설계하고 20을 test로 하여 예측을 해보는데 test error 가 가장 적게 나온 모델구하기

trainging set: parameter optimization
예) 모델을 가지고 기울기 판단하기

validation set: avoid overfitting
예) 100 개중 80개로 모델을 세우기, 단 너무 복잡한 validation set 은 오히려 test set 을 증가 시킬수 있다.

test set: measure the performance 예) 100 개중 아직 못본 20개 에 대해 판단하기, traning set 을 통해 test set 이 적어 지는 순간까지 구하기

Non parametric model

  • 모델(parametric model) 없이 알고리즘으로 찾아주는(성능 좋음)
  • 많은 samples 가 필요.
  • 사용하는 대표적인 방법
    • Kernel methods, input space(2차원)의 origin data를 계산식을 통한 feature space(3차원) 에 맵핑하여 구하기(좀 더 정확해 진다)
    • Gaussian Process Regression(GPR), regression, 데이터가 많지 않을 때
    • Suppert Vector Machine(SVM), classification

Suppert Vector Machine(SVM)

두 개의 class 를 구분하기 위한 좋은 선 찾기

kernel function 을 통해 feature space 에서 maximum margin 을 구해 linear classifiers 하기

예) 사람의 움직임의 신호(전기 동작)을 통해 패턴을 분석하여 사물에게 신호를 보내기

Gaussian Process Regression(GPR)

어떤 실수를 예측하기

관측된 점을 통해 함수를 통해 정규분포를 만들고 점마다 차원을 만들고 가우시안 분포를 생성한다.

입력마다 차원을 추가하여 모델을 생성하고 예측은 준비된 모델과 입력된 값을 join 하여 결과를 나타낸다.

데이터가 많으면 연산이 느려지는 큰 단점이 있다.

Dimension Reduction

low dim(2D) -> high dim(3D)

original space(high dim) -> feature space(low dim)

차원이 많으면 경험이 많기 필요하다.

PCA(Principal Component Analysis): 다 각도에서 변하는 수치중 큰 값만 찾기

9차원 vactor -> 1 차원 vactor 로 구할 수 있다.

Summary

  • Machine Leaning: Tasks 를 Experience 통해 Performance(성능) 향상에 목적.
    • Tasks:
    • Classification: 값이 구분되어 있는 경우
    • Regression: 연속된 값을 예측하는 경우
    • Clustring: (unsupervised) 목적 값이 없고 데이터 분석으로 하는 경우
    • etc
    • Performance
    • 0-1 loss: 목적을 숫자로 나타내기, 실패(=0), 성공(=1)
    • L2 loss: 목적의 근사치를 나태내기
    • etc
    • Experience
    • labeled data: (사진, 이름)등, 사용하기 쉽다
    • unlabelled data:
  • Machine Learning Process
    1. Select a parametric(먼저 데이터를 1, 2차 함수 일거다라고 예측) & nonparametric model
    2. regularization term(parametric model 생성시 너무 복잡해 질 수 있으므로 정규화하기)
    3. validation error & test set (전체중 모델과 작을 일부를 나눠 테스트를 하는 것)
  • Nonparametric model
    • Support Vector Machine
    • Gaussian Process Regression
  • Dimension reduction: PCA
