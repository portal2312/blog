정규화란 모델의 복잡도에 페널티를 줌으로써 과적합을 줄이는 것입니다.

  • 복잡도와 일반화 가능성 간의 절충에 대해 알아보기
  • L2 정규화 실험

학습 세트 가 올라가는 경우: 과적합, 익숙해 지기 시작함

ex) 테스트 세트 기준으로 손실을 줄이게 되므로, 그 외의 일반적으로 처리하기 힘듬

과적합을 피하기 위해 정규화라는 방법을 사용.

여러 전략중

조기 중단: 학습 데이터에 수렴하기 전에 학습을 멈춤

빨간 곡선(과적합)의 진짜 원인을 찾아 내려는 방법

모델 복잡도에 페널티 부여

  • 가능하면 모델 복잡도를 방지하려고 합니다.
  • 학습 단계에서 수행하는 최적화에 이 아이디어를 적용할 수 있습니다.

  • 경험적 위험 최소화

    • 학습 오류를 낮추는 것이 목표 최소화 손실 데이터 모델

  • 구조적 위험 최소화

    • 학습 오류를 낮추는 것이 목표
    • 또한 복잡도를 낮출 수 있도록 조정 최소화 손실 데이터 모델 & 복잡도 모델

정규화

  • 복잡도(모델)를 정의하는 방법
  • 더 작은 가중치 선호
  • 여기에서 벗어나면 비용이 발생함
  • L2 정규화(일명 능선)를 통해 이 아이디어를 구현할 수 있음
    • 복잡도 (모델) = 가중치의 제곱의 합
    • 아주 큰 가중치에 대한 페널티 부여
    • 선형 모델에서는 더 평평한 기울기를 선호
    • 베이지안 사전 확률:
    • 가중치는 0을 중심으로 배치되어야 함
    • 가중치는 정규 분포되어야 함
