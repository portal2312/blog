실제 ML 시스템 - 문학

이 강의에서는 18세기 문학과 관련된 실제 ML 문제*를 디버그합니다.

학습 목표

  • 실제의 ML 실험적 디자인에서 결함을 파악합니다.

실제 예: 18세기 문학

18세기 문학을 연구하는 교수가 작가들이 사용한 '마음에 대한 은유'만을 토대로 작가의 정치적 소속이 어디인지를 예측하고자 했습니다.

이 교수님은 18세기에 나온 모든 문학작품을 살펴보면서 저자가 '마음은 정원이다', '마음은 빈 서판이다' 등의 표현을 사용할 때마다 이를 기록했습니다. 은유적 표현에서 저자의 정치적 성향이 드러나는지 궁금해했죠.

저희는 우선 학습 및 시험용 데이터를 만들었습니다. 각 사례는 저자가 사용한 마음에 관한 은유적 표현이었으며, 라벨에는 휘그당, 토리당 등 18세기의 저자의 정치적 성향을 추가했습니다.

실제로 마음에 관한 은유적 표현만으로 저자의 정치적 성향을 예측할 수 있을 것이라 기대했습니다. 완벽한 성공이었죠. 그리고 정확성이 사실 의심스러울 정도로 높다는 사실을 깨달았습니다.

저희는 시험을 위한 학습용 및 검증용 세트를 나눌 때 사례를 문장별로 나눴다는 사실에 주목했습니다.

이에 따라, Samuel L. Richardson을 예로 들자면, 이 저자의 문장이 학습 데이터, 검증 데이터, 시험 데이터에 뿔뿔이 흩어지게 된 것입니다. 이는 저희가 사용한 모델이 Richardson이 사용한 은유뿐만 아니라 그의 언어 사용에서 보이는 특성을 학습하는 능력이 있었음을 의미합니다. 그러므로 어떤 의미에서는 시험에 적용해야 할 때, Richardson에 관한 추가 지식을 기억하고 있었다고 말할 수 있죠.

{
  "학습 세트": ["Swift", "Blake", "Defoe"],
  "검증 세트": ["Swift", "Blake", "Defoe"],
  "테스트 세트": ["Swift", "Blake", "Defoe"]
}

그래서 저희는 다시 실험을 수행했습니다. 이번에는 사례를 저자별로 나눠 특정 저자는 학습, 시험, 검증 데이터 중 한 곳에만 속하게 되었습니다.

{
  "학습 세트": ["모든 Swift 사례"],
  "검증 세트": ["모든 Blake 사례"],
  "테스트 세트": ["모든 Defoe 사례"]
}

이번 실험 결과 시험 데이터에서 좋은 정확성을 얻기가 훨씬 더 힘들었습니다. 또한 은유 표현과 관련된 데이터만 기반으로 하여 정치적인 성향을 예측하기도 훨씬 어려워졌습니다. 흥미로운 작업이었고 각각의 관점에서 두 개의 논문을 작성할 수 있었다.

또 학습용 데이터와 시험용 데이터를 어떻게 나눌지 고민하는 것이 얼마나 중요한지도 깨달았습니다.

머신러닝을 다루는 사람은 '음, 데이터를 정규화하면 되지.'라고 생각해버리기 쉽습니다. 하지만 데이터를 잘 나누기 위해서는 이 데이터가 무엇을 뜻하는지 이해해야 합니다.
