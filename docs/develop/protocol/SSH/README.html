SSH Connection

Options

  -p      Port
  -v      Debug mode, flow 를 볼 수 있음, v를 붙이면 자세히 볼 수 있음 (-vvv 까지)

Example

> ssh -p 22 -v root@server_ip

SSH key-gen

Options

  -t type     Specify type of key to create.

Example

> ssh-keygen -t rsa
Generating public/private rsa key pair.
Enter file in which to save the key (/root/.ssh/id_rsa): 저장할 경로
/root/.ssh/id_rsa already exists.
Overwrite (y/n)? y
Enter passphrase (empty for no passphrase): 암호 추가, SSH 접속시 물어봄, 보안강화
Enter same passphrase again:
Your identification has been saved in /root/.ssh/id_rsa.
Your public key has been saved in /root/.ssh/id_rsa.pub.
The key fingerprint is:
1b:c4:68:a4:88:96:bb:75:0e:b0:a5:c3:46:c7:20:32 root@localhost.localdomain
The key's randomart image is:
+--[ RSA 2048]----+
|E.    .          |
|oo+. o o         |
|.=.+. o o        |
|+ B  . .         |
| B o .  S        |
|. + +    o       |
| .   .  .        |
|                 |
|                 |
+-----------------+

완료후 생성된 파일 목록

> cd .ssh
> ls -al
id_ras       # Private-Key, 보안상 노출하면 안됨
id_rsa.pub   # Public-Key
known_hosts  # 알 수 없는 시스템 목록이 표현
authorized_keys  # Server 측 파일에서 Client의 id_rsa.pub 값 저장시 접속 허용함

자동 로그인 설정

  cat ~/.ssh/id_rsa.pub | ssh root@server_ip 'cat >> ~/.ssh/authorized_keys'

sshd_config

openssh-server 설정 파일이라고도 불림.

$ su -
$ vi /etc/ssh/sshd_config

설정 파일에 대한 자세한 내용

# /etc/ssh/sshd_config

Port 22  # XXX: 설치되어 있다하더라고 주석을 해제해야 접속가능

#PermitRootLogin prohibit-password
PermitRootLogin yes  # XXX: root 접속 허용하기(위 내용 복사후 yes로 변경)

재기동 해야 적용 됨

$ su -
$ systemctl restart sshd.service
