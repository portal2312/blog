root 권한의 계정으로 실행하기

install 당시의 예: ldap.mk.com

Admin

Modify Password (Hashing Password)

비밀번호 SSHA-1 으로 만든 후 복사하기:

slappasswd -h {SSHA}

mod_admin_passwd.ldif 파일 작성하기:

dn: olcDatabase={1}hdb,cn=config
#olcRootDN: cn=admin,dc=ldap,dc=mk,dc=com
changetype: modify
replace: olcRootPW
olcRootPW: {SSHA}tuIwal2NMYnwz9mCjcY7OgKQFj5lTYsq

$ ldapmodify -H ldapi:// -Y EXTERNAL -f ~/newpasswd.ldif

SASL/EXTERNAL authentication started
SASL username: gidNumber=0+uidNumber=0,cn=peercred,cn=external,cn=auth
SASL SSF: 0
modifying entry "olcDatabase={1}hdb,cn=config"

dn: ou=People,dc=example,dc=com
objectClass: organizationalUnit
ou: People

dn: ou=Groups,dc=example,dc=com

dn: cn=miners,ou=Groups,dc=example,dc=com
objectClass: posixGroup

Users

Add

파일명.ldif 작성후 ldapadd 명령어 실행하기:

dn: uid=john,ou=People,dc=example,dc=com
objectClass: inetOrgPerson
objectClass: posixAccount
objectClass: shadowAccount
uid: john
sn: Doe
givenName: John
cn: John Doe
displayName: John Doe
uidNumber: 10000
gidNumber: 5000
userPassword: johnldap
gecos: John Doe
loginShell: /bin/bash
homeDirectory: /home/john

$ ldapadd -x -D "cn=admin,dc=ldap,dc=mk,dc=com" -W -f 파일명.ldif

Enter LDAP Password:
adding new entry "uid=john,ou=People,dc=ldap,dc=mk,dc=com"

Remove

ldif 없이 즉시 실행하기:

$ ldapdelete -x -D cn=admin,dc=ldap,dc=mk,dc=com -W uid=mkkim,ou=People,dc=ldap,dc=mk,dc=com

Modify

파일명.ldif 작성후 ldapmodify 명령어 실행하기:

$ ldapmodify -x -D "cn=admin,dc=ldap,dc=mk,dc=com" -W -f 파일명.ldif

Add attribute:

dn: uid=john,ou=People,dc=ldap,dc=mk,dc=com
changetype: modify
add: mail
mail: mkkim@zensystems.co.kr

Replace attribute:

dn: uid=john,ou=People,dc=ldap,dc=mk,dc=com
changetype: modify
replace: mail
mail: mkkim@zensystems.com

Delete attribute:

dn: uid=john,ou=People,dc=ldap,dc=mk,dc=com
changetype: modify
delete: mail

attribute: value 쌍을 지정하지 않고 delete 구문을 사용하면 이 속성의 모든 값이 제거됩니다. attribute: value 쌍을 지정하면 해당 값만 제거됩니다.

Password

$ ldappasswd -H ldap://server_domain_or_IP -x -D "user_dn" -W -A -S

Search

$ ldapsearch -x -b ou=People,dc=ldap,dc=mk,dc=com
