Linux: ubuntu

Repository Server List

$ vi /etc/apt/sources.lists

# deb cdrom:[Ubuntu 10.10 _Maverick Meerkat_ - Release i386 (20101007)]/ maverick main restricted
# See http://help.ubuntu.com/community/UpgradeNotes for how to upgrade to
# newer versions of the distribution.
deb http://kr.archive.ubuntu.com/ubuntu/ natty main restricted
deb-src http://kr.archive.ubuntu.com/ubuntu/ natty main restricted
## Major bug fix updates produced after the final release of the
## distribution.
deb http://kr.archive.ubuntu.com/ubuntu/ natty-updates main restricted
deb-src http://kr.archive.ubuntu.com/ubuntu/ natty-updates main restricted
# ...

```bash

deb

미리 컴파일된 바이너리를 포함한 패키지를 관리하는 저장소라는 것을 의미. 대부분 이형식을 사용.

deb-src

컴파일하지 않은 소스 코드를 포함한 패키지를 관리하는 저장소라는 것을 의미. 소스를 분석할 일이 많은 개발자가 주로 사용.

natty

저장소에서 관리하는 배포판의 버전 이름.

main, restricted

섹션 이름. ```

저장소 추가시 apt-get update 으로 갱신.

Usage

# XXX: Install
$ apt-get install <package-name>
$ apt-get --reinstsall install  # XXX: 재설치

# XXX: upgrade
$ apt-get upgrade <package-name>
$ apt-get dist-upgrade <package-name>  # XXX: 의존성

# XXX: Delete
$ apt-get remove <package-name>
$ apt-get --purge remove <package-name>  # XXX: 설정파일까지

# XXX: Search
$ apt-cache search <str> | grep <str>
$ apt-cache show <str>  # XXX: 정보 보기
$ dpkg -l <str> | grep <xtr>  # XXX: 설치된 목록 보기

$ apt-get source <package-name>  # XXX: 소스 다운로드
$ apt-get build-dep <package-name>  # XXX: 받은 소스 의존성 빌드하기
$ apt-get clean
$ apt-get autoclean

sources.list

$ vi /etc/apt/sources.list

# fixed & save: fpt.daumkakao.com

$ apt-get update

Exception

software OS package version update error (17.4 -> 17.10)

소프트웨어 업데이트 시 Erorr

해결:

$ su -
$ vi etc/apt/sources.list
# packages repository 변경하기, fpt.daumkakao.com 로 변경하면 빠르다
$ sources.list update  # 적용 및 갱신
# 다시 소프트웨어 업데이트 실행

소프트웨어 업데이트 진행후 background 에서 dpkg 로 작업된다.

절대 shutdown 하지 하지 말고 top 명령어로 dpkg process 확인해서 없어지면 shutdown -r now 하기. (task manager 에서는 dpkg process 가 없었다)

shotdown 이후 아래 작업을 한다.

  • 한글 key
  • 기본 packages (usb 자동마운트 등)
  • 사용자 설치에 의한 packages
$ apt-get upgrade  # 한글 key, 기본 packages 호환성 해결

VirtualBox 는 새로운 version 의 *.deb 설치하면 해결된다.
