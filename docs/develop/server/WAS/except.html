설정 파일

/usr/local/etc/uwsgi/*.ini

내용

[uwsgi]
uid=zen
gid=zen
python-path=/home/zen/virtualenvs/zenlog/virenv361/lib/python3.6
python-path=/home/zen/virtualenvs/zenlog/virenv361/lib/python3.6/site-packages
logger=file:/opt/zen/logs/exc/uwsgi.debug
chdir=/opt/zen/webapps/ddns_portal
module=ddns.wsgi:application
env=DJANGO_SETTINGS_MODULE=ddns.settings
post-buffering=819200
socket=/run/uwsgi/ddns_portal.sock
processes=4
master=1
listen=1024
max-requests=1000
vacuum=true
enable-threads=true
close-on-exec=true
harakiri=181
harakiri-verbose=true
disable-logging=true
catch-exceptions=true

테스트

uwsgi --ini ddns_portal.ini

systemctl

$ cd /etc/systemd/system/multi-user.target.wants
$ ls
uwsgi@name1.service uwsgi@name2.service

원래 파일:

# /lib/systemd/system/uwsgi@service
[Unit]
Description=uWSGI instance unit
After=network.target

[Service]
User=zen
Group=zen
ExecStartPre=/bin/mkdir -p /run/uwsgi
ExecStart=/usr/local/bin/uwsgi --ini /usr/local/etc/uwsgi/%I.ini
ExecReload=/bin/kill -HUP $MAINPID
ExecStop=/bin/kill -INT $MAINPID
Restart=always
Type=notify
StandardError=syslog
KillSignal=SIGQUIT
RuntimeDirectory=uwsgi
RuntimeDirectoryMode=755


[Install]
WantedBy=multi-user.target

추가된 파일:

# /etc/systemd/system/multi-user.target.wants/uwsgi@ddns_portal.service
[Unit]
Description=uWSGI DDNS portal
After=network.target

[Service]
User=zen
Group=zen
ExecStartPre=/bin/mkdir -p /run/uwsgi
ExecStart=/usr/local/bin/uwsgi --ini /usr/local/etc/uwsgi/ddns_portal.ini
ExecReload=/bin/kill -HUP $MAINPID
ExecStop=/bin/kill -INT $MAINPID
Restart=always
Type=notify
StandardError=syslog
KillSignal=SIGQUIT
RuntimeDirectory=uwsgi
RuntimeDirectoryMode=755


[Install]
WantedBy=multi-user.target

Listen queue size is greater than the system max

uwsgi 재기동시 아래와 같은 메시지인 경우

uwsgi.log:

thunder lock: disabled (you can enable it with --thunder-lock)
*** RRDtool library available at 0x176a760 ***
Listen queue size is greater than the system max net.core.somaxconn (128).

커널 파라미터가 listen backlog, 즉 listen()으로 바인딩 된 서버 소켓에서 accept()를 기다리는 소켓 개수에 관련된 커널 파라미터는 'net.core.somaxconn'입니다.

이 값은 listen() 시스템 콜의 매개변수로 설정하는 backlog 값의 hard limit입니다. 서버 애플리케이션에서 listen()시 적당히 설정해야겠지만, 먼저 이 hard limit을 증가시켜야 할 것입니다.

$ su -
$ sysctl -w net.core.somaxconn="1024"

uwsgi.ini 알맞게 수정하기:

post-buffering=521
socket=/tmp/uwsgi.sock;
processes=16
master=1
listen=256
max-requests=256
