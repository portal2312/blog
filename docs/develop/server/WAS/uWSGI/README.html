uWSGI(Web Server Gateway Interface)

Table of contents

  • Included components (updated to latest stable release)
  • Reference

uWSGI 프로젝트는 호스팅 서비스 구축을위한 전체 스택을 개발하는 것을 목표로합니다.

응용 프로그램 서버 (다양한 프로그래밍 언어 및 프로토콜 용), 프록시, 프로세스 관리자 및 모니터는 모두 공통 API 및 공통 구성 스타일을 사용하여 구현됩니다.

플러그 형 아키텍처로 인해 더 많은 플랫폼과 언어를 지원하도록 확장 될 수 있습니다. 현재 C, C ++ 및 Objective-C로 플러그인을 작성할 수 있습니다. 그 이름의 "WSGI"부분은 프로젝트를위한 최초의 개발 된 플러그인이기 때문에 이름이있는 Python 표준에 대한 찬사입니다. 다 기능성, 성능, 낮은 리소스 사용 및 안정성은 프로젝트의 강점입니다 (그리고 뒤 따르는 유일한 규칙).

Included components (updated to latest stable release)

Core (구성, 프로세스 관리, 소켓 생성, 모니터링, 로깅, 공유 메모리 영역, ipc, 클러스터 멤버쉽 및 uWSGI Subscription Server 구현)

Request plugins (다양한 언어 및 플랫폼 용 응용 프로그램 서버 인터페이스 구현 : WSGI, PSGI, Rack, Lua WSAPI, CGI, PHP, Go ...)

Gateways (로드 밸런서, 프록시 및 라우터 구현)

Emperor (방대한 인스턴스 관리 및 모니터링 구현)

루프 엔진 (사전 처리, 스레드, 비동기/이벤트 및 녹색 스레드/코 루틴 모드에서 구성 요소를 실행할 수 있습니다. uGreen, Greenlet, Stackless, Gevent, Coro 등 다양한 기술이 지원됩니다::AnyEvent, Tornado, goroutines 및 섬유)

NOTE

uWSGI는 빠른 출시주기를 가진 매우 활동적인 프로젝트입니다.
이러한 이유로 코드와 설명서가 항상 동기화되지 않을 수 있습니다.
(생략)
