Installation

For Windows

pycharm-x.x.x.exe 실행하기

For Linux

  1. 설치 파일 다운로드 (pycharm-x-x-x.tar.gz)
  2. 설치 bash $ tar xzvfp pycharm-x.x.x.tar.gz $ cd ./pycharm-x.x.x $ vi ./Install-Linux-tar.txt # XXX: 설치에 대한 상세 내용 (넘어가도 됨) $ cd ./bin $ ./pycharm.sh
  3. 설치 중 실행파일을 script 파일로 만들겠다 라는 부분에 check (기본 경로: /usr/local/bin/charm)

  4. pycharm 바로가기 만들기 (3번에서 수정했다면 pass) bash $ su - $ cd /usr/local/bin $ ln -s charm pycharm

    /usr/local/bin/charm 등록시:

    root 계정에서 설치후 일반 계정에서 실행하면 passwd 를 물어본다.

    일반 계정에서 설치후 일반 계정에서 실행하면 pycharm install 과정중에 passwd 한번만 입력하면 추후 실행시 다시 묻지 않는다.

    1. 바로가기가 자동생성된 icon 을 편집해서 실행경로를 맞춰주면 자연스럽게 background process 로 사용가능 하다.

Memory config

Editor 전체적으로 버퍼가 부족해서 느려지는 경우 수정.

Menu: Help > Edit Custom VM options

직접 수정도 가능하지만 pycharm을 동작한 계정별 vmoption이라 menu에서 설정하도록 하자.

# custom PyCharm VM options

-Xms2048m  # 최소 memory
-Xmx4096m  # 최대 memory
-XX:ReservedCodeCacheSize=1024m  # 재시작시 소스코드 가져오는 캐쉬 크기
-XX:+UseConcMarkSweepGC
-XX:SoftRefLRUPolicyMSPerMB=50
-ea
-Dsun.io.useCanonCaches=false
-Djava.net.preferIPv4Stack=true
-XX:+HeapDumpOnOutOfMemoryError
-XX:-OmitStackTraceInFastThrow
-Dawt.useSystemAAFontSettings=lcd
-Dsun.java2d.renderer=sun.java2d.marlin.MarlinRenderingEngine

Config info

$ vi /usr/local/bin/pycharm

# ...
CONFIG_PATH = u'/root/.PyCharm2017.1/config'
# ...

:q (종료)

$ cd ~/.PyCharm2017.1/config

Tools

Run/Debug Configurations

host: 0.0.0.0 / port: 8000

Environment

Environment variables: Project의 settings.py 를 설정, 보통 자동으로 됨

Except

Error running "Django-server Name": please select Django module

Projects 불러올 경우 보통 자동으로 설정이 잡히나 안되는 경우가 있음.

Project가 django-server 사용하고 아래와 같이 설정되어 있다면

Run/Debug Configurations

...
Environment variable: PYTHONUNBUFFERED=1

Environment variable 옆의 +로 아래와 같이 추가한다.

Name Value
PYTHONUNBUFFERED 1
DJANGO_SETTINGS_MODULE .settings

추가해도 안될 경우 source를 다시 받고 indexing 완료 될 때 까지 다시 기다리자.

Uninstall

Linux:

$ rm -rf ~/.Pycharm{Version}
